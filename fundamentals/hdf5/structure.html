HDF5 Structure

Understanding the HDF5 Structure in BigDataStatMeth

An HDF5 file behaves like a miniature file system inside a single file, where data is organized into groups, datasets, and attributes. In BigDataStatMeth, the user decides the hierarchy: the package does not impose a fixed structure, but automatically creates any missing file, group, or dataset.

Tip

Key concept — You define the structure that best fits your project.
BigDataStatMeth takes care of creating the file, group, and dataset if they do not exist.

The diagram above illustrates a simple hierarchy automatically created by bdCreate_hdf5_matrix(): a file (demo.h5) containing one group (/myData) and one dataset (X). Each level of the hierarchy is created automatically if missing.

Components of an HDF5 File

An HDF5 file is composed of three fundamental building blocks:

TipGroups
  • Act like folders in a filesystem
  • Contain datasets or other groups
  • Help organize data hierarchically
TipDatasets
  • Multidimensional arrays (numeric, text, etc.)
  • Store matrix-like data (e.g., X, Y, Z)
  • Accessed directly or by blocks in BigDataStatMeth
TipAttributes
  • Metadata describing datasets or groups
  • Small key–value pairs (units, sample names, etc.)
  • Improve reproducibility and self-description

Example in R

library(BigDataStatMeth)

X <- matrix(rnorm(12), nrow = 3)
bdCreate_hdf5_matrix("demo.h5", X,
                     group = "myData",
                     dataset = "X",
                     overwriteFile = TRUE)
rhdf5::h5ls("demo.h5")
    group   name       otype dclass   dim
0       / myData   H5I_GROUP             
1 /myData      X H5I_DATASET  FLOAT 3 x 4

This command creates a new HDF5 file named demo.h5 (if it does not already exist) and stores a dataset X within a group called myData.
The structure can be inspected with rhdf5::h5ls().

Equivalent C++ Example (not executed)

#include "BigDataStatMeth.h"
int main() {
  BigDataStatMeth::hdf5File file("demo.h5", true);
  file.createFile();

  BigDataStatMeth::hdf5Dataset dataset(&file, "myData", "X", true);
  dataset.createDataset(3, 4, "numeric");

  Rcpp::NumericMatrix X = Rcpp::rnorm(12);
  dataset.writeDataset(X);
  return 0;
}
Note

Note — The R function bdCreate_hdf5_matrix() performs these same C++ steps internally: it creates the file, the group, and the dataset, then writes the data automatically.

Proposed structure for BigDataStatMeth analyses

Although the user can freely define their own hierarchy, the following layout is a recommended structure that helps maintain a clear and reproducible organization across analyses.

Group Example datasets Description
/data X_raw, X_scaled Input matrices or intermediate blocks
/meta sample_info, variables Metadata and annotations
/results SVD, CCA Outputs and computed results
Tip

Good practice — Organize files using clear and consistent group names (e.g., /data, /meta, /results) to make your analyses easier to navigate and reproduce.

Summary

  • HDF5 organizes information into groups, datasets, and attributes.
  • In BigDataStatMeth, users freely define their hierarchy.
  • The package automatically creates missing elements when writing.
  • A structured organization improves clarity, collaboration, and reproducibility.
