Quick Overview

Noteüìã TL;DR - Essential Best Practices

Data Organization: - Plan HDF5 structure before creating - Use descriptive names for groups/datasets - Separate raw from processed data - Document with attributes (metadata)

BigDataStatMeth Workflow: - Create ‚Üí Process ‚Üí Analyze ‚Üí Inspect - Let BigDataStatMeth manage performance - Use parallelization for large data - Results automatically organized in HDF5

Performance: - Use default settings (already optimized) - Enable paral=TRUE for large datasets - Set appropriate threads count - Monitor with simple benchmarks

Data Safety: - Keep raw data unchanged - Process into new datasets - Test on small data first - Back up critical files

Workflow Guidelines

1. Planning Your Analysis

Start with a workflow diagram

# analysis_plan.yaml
project: genomic_pca_analysis
input: genotype_data.csv
output: analysis_results.hdf5

workflow:
  1_import:
    - Load genotype data
    - Create HDF5 file
  2_qc:
    - Remove high-missing SNPs
    - Filter by MAF
  3_preprocessing:
    - Impute missing values
    - Normalize data
  4_analysis:
    - Compute PCA
    - Calculate correlations
  5_results:
    - Extract components
    - Visualize

Design HDF5 structure upfront

2. Standard BigDataStatMeth Workflow

Complete analysis template

library(BigDataStatMeth)
library(rhdf5)

# Configuration
filename <- "analysis.hdf5"
n_threads <- 4

# ============================================================
# STEP 1: IMPORT DATA
# ============================================================

# From R object
genotypes <- read.csv("genotypes.csv")

bdCreate_hdf5_matrix(
  filename = filename,
  object = as.matrix(genotypes),
  group = "data/raw",
  dataset = "genotypes",
  overwriteFile = TRUE
)

cat("‚úì Data imported\n")

# ============================================================
# STEP 2: QUALITY CONTROL
# ============================================================

# Remove SNPs with >5% missing
bdRemovelowdata_hdf5(
  filename = filename,
  group = "data/raw",
  dataset = "genotypes",
  outgroup = "data/processed",
  outdataset = "qc_step1",
  bycols = TRUE,
  pcent = 0.05
)

# Filter by MAF
bdRemoveMAF_hdf5(
  filename = filename,
  group = "data/processed",
  dataset = "qc_step1",
  outgroup = "data/processed",
  outdataset = "qc_filtered",
  maf = 0.05,
  overwrite = TRUE
)

cat("‚úì QC completed\n")

# ============================================================
# STEP 3: PREPROCESSING
# ============================================================

# Impute missing SNPs
bdImputeSNPs_hdf5(
  filename = filename,
  group = "data/processed",
  dataset = "qc_filtered",
  bycols = TRUE,
  overwrite = TRUE
)

# Normalize data
bdNormalize_hdf5(
  filename = filename,
  group = "data/processed",
  dataset = "qc_filtered",
  outgroup = "data/processed",
  outdataset = "normalized",
  bcenter = TRUE,
  bscale = TRUE
)

cat("‚úì Preprocessing completed\n")

# ============================================================
# STEP 4: STATISTICAL ANALYSIS
# ============================================================

# PCA
pca_result <- bdPCA_hdf5(
  filename = filename,
  group = "data/processed",
  dataset = "normalized",
  k = 20,
  bcenter = FALSE,  # Already centered
  bscale = FALSE,   # Already scaled
  threads = n_threads,
  paral = TRUE
)

cat("‚úì PCA completed\n")

# Correlation matrix
bdCorr_hdf5(
  filename = filename,
  group = "data/processed",
  dataset = "normalized"
)

cat("‚úì Analysis completed\n")

# ============================================================
# STEP 5: INSPECT RESULTS
# ============================================================

# View file structure
cat("\nFile structure:\n")
h5ls(filename)

# Read PCA components
components <- h5read(filename, "/PCA/normalized/components")
eigenvalues <- h5read(filename, "/PCA/normalized/eigenvalues")

# Calculate variance explained
variance_explained <- eigenvalues^2 / sum(eigenvalues^2)
cat("\nVariance explained by first 5 PCs:\n")
print(round(variance_explained[1:5] * 100, 2))

# Clean up
h5closeAll()

cat("\n‚úì Workflow completed successfully\n")

3. Naming Conventions

Recommended naming scheme

Groups: lowercase with underscores

/data/raw_genotypes/
/data/processed_data/
/analysis/quality_control/

Datasets: descriptive, lowercase

genotype_matrix
phenotype_data
pca_components
correlation_matrix

Consistent prefixes:

qc_filtered      # After quality control
imputed_data     # After imputation
normalized_expr  # After normalization
pca_components   # PCA results

4. Documentation with Metadata

Add metadata to datasets

BigDataStatMeth doesn‚Äôt have built-in attribute functions, so use rhdf5:

library(rhdf5)

# After creating/processing dataset
h5writeAttribute(
  attr = "Quality-controlled genotype data",
  file = filename,
  name = "data/processed/qc_filtered",
  attrname = "description"
)

h5writeAttribute(
  attr = list(
    maf_threshold = 0.05,
    missing_threshold = 0.05,
    date = as.character(Sys.Date()),
    software = "BigDataStatMeth",
    version = as.character(packageVersion("BigDataStatMeth"))
  ),
  file = filename,
  name = "data/processed/qc_filtered",
  attrname = "processing_info"
)

h5closeAll()

Read metadata

# Read all attributes
attrs <- h5readAttributes(filename, "data/processed/qc_filtered")
print(attrs)

h5closeAll()

Production Patterns

Pattern 1: Incremental Processing

Process large datasets incrementally:

# Process genomic data by chromosome
chromosomes <- 1:22

filename <- "genome_wide.hdf5"

for (chr in chromosomes) {
  cat("Processing chromosome", chr, "\n")
  
  # Load chromosome data
  chr_data <- load_chromosome(chr)
  
  # Add to HDF5
  bdCreate_hdf5_matrix(
    filename = filename,
    object = chr_data,
    group = "chromosomes",
    dataset = paste0("chr", chr),
    overwriteFile = (chr == 1)  # Create file on first chr
  )
  
  # QC for this chromosome
  bdRemovelowdata_hdf5(
    filename = filename,
    group = "chromosomes",
    dataset = paste0("chr", chr),
    outgroup = "qc_chromosomes",
    outdataset = paste0("chr", chr, "_qc"),
    bycols = TRUE,
    pcent = 0.05
  )
}

# Combine all QC'd chromosomes
bdBind_hdf5_datasets(
  filename = filename,
  group = "qc_chromosomes",
  datasets = paste0("chr", 1:22, "_qc"),
  outgroup = "genome_wide",
  outdataset = "all_snps",
  by_rows = FALSE  # Bind by columns (SNPs)
)

Pattern 2: Reproducible Analysis

Create reproducible analysis scripts:

#!/usr/bin/env Rscript
# reproducible_analysis.R

# ============================================================
# Configuration
# ============================================================

CONFIG <- list(
  input_file = "raw_data.csv",
  output_file = "analysis_results.hdf5",
  qc = list(
    missing_threshold = 0.05,
    maf_threshold = 0.05
  ),
  pca = list(
    n_components = 20,
    center = TRUE,
    scale = FALSE
  ),
  parallel = list(
    enabled = TRUE,
    threads = 4
  )
)

# ============================================================
# Functions
# ============================================================

run_qc <- function(filename, config) {
  cat("Running QC...\n")
  
  bdRemovelowdata_hdf5(
    filename = filename,
    group = "data/raw",
    dataset = "input",
    outgroup = "data/qc",
    outdataset = "step1",
    bycols = TRUE,
    pcent = config$qc$missing_threshold
  )
  
  bdRemoveMAF_hdf5(
    filename = filename,
    group = "data/qc",
    dataset = "step1",
    outgroup = "data/qc",
    outdataset = "filtered",
    maf = config$qc$maf_threshold,
    overwrite = TRUE
  )
}

run_pca <- function(filename, config) {
  cat("Running PCA...\n")
  
  bdPCA_hdf5(
    filename = filename,
    group = "data/qc",
    dataset = "filtered",
    k = config$pca$n_components,
    bcenter = config$pca$center,
    bscale = config$pca$scale,
    threads = config$parallel$threads,
    paral = config$parallel$enabled
  )
}

# ============================================================
# Main
# ============================================================

main <- function() {
  library(BigDataStatMeth)
  
  # Import data
  cat("Importing data...\n")
  data <- read.csv(CONFIG$input_file)
  
  bdCreate_hdf5_matrix(
    filename = CONFIG$output_file,
    object = as.matrix(data),
    group = "data/raw",
    dataset = "input",
    overwriteFile = TRUE
  )
  
  # Run pipeline
  run_qc(CONFIG$output_file, CONFIG)
  run_pca(CONFIG$output_file, CONFIG)
  
  # Save config
  library(rhdf5)
  h5writeAttribute(
    attr = jsonlite::toJSON(CONFIG, pretty = TRUE),
    file = CONFIG$output_file,
    name = "/",
    attrname = "analysis_config"
  )
  h5closeAll()
  
  cat("Analysis completed: ", CONFIG$output_file, "\n")
}

# Run
if (!interactive()) {
  main()
}

Pattern 3: Testing on Subsets

Test workflow on small subsets first:

# Development: Test on 1000 samples
dev_workflow <- function() {
  # Load full data
  full_data <- load_data("full_dataset.csv")
  
  # Create test subset
  test_data <- full_data[1:1000, 1:5000]  # 1000 samples, 5000 features

    filename = "test_analysis.hdf5",

}

}

Pattern 4: Parallel-Safe Workflows

Design workflows for parallel execution:

# Multiple R sessions can read simultaneously
library(parallel)

process_chunk <- function(chunk_id) {
  library(BigDataStatMeth)
  
  filename <- "shared_data.hdf5"
  
  # Each process works on different data
  start_col <- (chunk_id - 1) * 1000 + 1
  end_col <- chunk_id * 1000
  
  # Extract subset
  bdsubset_hdf5_dataset(
    filename = filename,
    group = "data",
    dataset = "full_matrix",
    rows = NULL,
    cols = start_col:end_col,
    outgroup = "chunks",
    outdataset = paste0("chunk_", chunk_id)
  )
  
  # Process chunk
  bdPCA_hdf5(
    filename = filename,
    group = "chunks",
    dataset = paste0("chunk_", chunk_id),
    k = 10
  )
  
  return(chunk_id)
}

# Run in parallel (safe: each writes to different location)
n_chunks <- 10
results <- mclapply(1:n_chunks, process_chunk, mc.cores = 4)

Common Pitfalls

Pitfall 1: Not Closing rhdf5 Handles

Warning‚ö†Ô∏è Always Close After Inspection

Problem:

# BAD: Using rhdf5 for inspection
library(rhdf5)
h5ls("file.hdf5")
# Forgot to close!

# Later... BigDataStatMeth function fails
bdPCA_hdf5("file.hdf5", ...)  # ERROR: File locked

Solution:

# GOOD: Always close after rhdf5
library(rhdf5)
h5ls("file.hdf5")
h5closeAll()  # ‚Üê CRITICAL

# Now BigDataStatMeth works
bdPCA_hdf5("file.hdf5", ...)

Best practice:

# Use tryCatch with finally
inspect_file <- function(filename) {
  tryCatch({
    library(rhdf5)
    structure <- h5ls(filename)
    print(structure)
  }, finally = {
    h5closeAll()  # Always runs
  })
}

Pitfall 2: Processing Without QC

Warning‚ö†Ô∏è Always Perform Quality Control

Problem:

# BAD: Direct analysis on raw data
bdPCA_hdf5(filename, "data", "raw_genotypes")
# Results biased by missing data and low-quality SNPs

Solution:

# GOOD: QC pipeline first
# 1. Remove high-missing
bdRemovelowdata_hdf5(...)

# 2. Filter MAF
bdRemoveMAF_hdf5(...)

# 3. Impute remaining missing
bdImputeSNPs_hdf5(...)

# 4. Then analyze
bdPCA_hdf5(filename, "data", "qc_filtered")

Pitfall 3: Overwriting Important Data

Warning‚ö†Ô∏è Preserve Raw Data

Problem:

# BAD: Overwriting original data
bdNormalize_hdf5(
  filename = filename,
  group = "data",
  dataset = "genotypes",
  outgroup = "data",
  outdataset = "genotypes",  # ‚Üê Overwrites!
  overwrite = TRUE
)

Solution:

# GOOD: Create new processed dataset
bdNormalize_hdf5(
  filename = filename,
  group = "data/raw",
  dataset = "genotypes",
  outgroup = "data/processed",  # Different location
  outdataset = "normalized",
  overwrite = FALSE
)

Pitfall 4: Inappropriate Parallelization

Warning‚ö†Ô∏è Use Parallelization Wisely

Problem:

# BAD: Over-parallelizing small data
small_data <- matrix(rnorm(1000), 100, 10)

bdPCA_hdf5(filename, group, dataset,
           paral = TRUE,
           threads = 32)  # Overhead > benefit

Solution:

# GOOD: Match parallelization to data size

# Small data (< 100 MB)
bdPCA_hdf5(filename, group, dataset,
           paral = FALSE)

# Medium data (100 MB - 10 GB)
bdPCA_hdf5(filename, group, dataset,
           paral = TRUE,
           threads = 4)

# Large data (> 10 GB)
bdPCA_hdf5(filename, group, dataset,
           paral = TRUE,
           threads = 16,
           q = 2)  # Two-level hierarchy

Pitfall 5: Ignoring File Structure

Warning‚ö†Ô∏è Organize Hierarchically

Problem:

# BAD: Flat structure
/genotypes
/genotypes_qc
/genotypes_qc_imputed
/genotypes_qc_imputed_normalized
/pca_result
/corr_result

Solution:

# GOOD: Hierarchical organization
/data/
  /raw/genotypes
  /qc/filtered
  /processed/normalized
/analysis/
  /pca/components
  /correlation/matrix

Summary

Best practices for BigDataStatMeth workflows:

Planning: - Design HDF5 structure upfront - Document analysis plan - Use consistent naming

Workflow: - Import ‚Üí QC ‚Üí Process ‚Üí Analyze ‚Üí Inspect - Keep raw data separate - Process into new datasets - Use appropriate parallelization

Safety: - Test on subsets first - Close rhdf5 handles - Never overwrite raw data - Back up critical files

Performance: - Use defaults (optimized) - Enable parallelization for large data - Monitor and benchmark - Use hierarchical processing when needed
