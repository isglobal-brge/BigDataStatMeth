Parallelism in BigDataStatMeth

Understanding parallelism

Modern datasets are massive. Operating on them efficiently requires parallel computation — splitting large tasks into smaller parts that can run simultaneously on multiple CPU threads.

BigDataStatMeth implements block-wise parallelism, distributing blocks of a matrix (or HDF5 dataset) among several threads to accelerate computation without exceeding available memory.

Tip

Key idea: Parallelism is achieved by dividing the matrix into blocks. Each thread handles one block (or a subset of blocks) independently, and the results are later combined.

Parallelism in R

In R, parallelism is typically achieved using packages like parallel, foreach, or BiocParallel. However, BigDataStatMeth integrates parallel support directly into its functions through the threads parameter.

Example: multiplying two matrices in memory using multiple threads.

  • R
library(BigDataStatMeth)

set.seed(1)
A <- matrix(rnorm(1e4), nrow = 100, ncol = 100)
B <- matrix(rnorm(1e4), nrow = 100, ncol = 100)

# Single-thread vs multi-thread multiplication
system.time(res1 <- bdblockMult(A, B, threads = 1))
   user  system elapsed 
  0.000   0.001   0.000 
system.time(res2 <- bdblockMult(A, B, threads = 4))
   user  system elapsed 
      0       0       0
ImportantInternal execution flow
  1. Split matrices A and B into small blocks
  2. Distribute blocks among available threads
  3. Compute partial results independently
  4. Combine results into the final matrix

As the number of threads increases, computation speed improves until the overhead of synchronization and memory access becomes dominant.

Parallelism in C++

The C++ API uses OpenMP for parallel processing, which provides fine-grained control and high performance.

Example pattern used internally by BigDataStatMeth:

#pragma omp parallel for num_threads(threads)
for (int i = 0; i < num_blocks; ++i) {
    auto A_block = read_hdf5(A, i);
    auto B_block = read_hdf5(B, i);
    auto C_block = multiply(A_block, B_block);
    write_hdf5(C, i, C_block);
}

Visual overview

Each thread computes a portion of the final result. Once all threads finish, the results are combined into the output matrix or HDF5 dataset.

WarningI/O bottleneck during parallel writes

When many threads write to the same HDF5 file simultaneously, disk contention can occur.
This can become a bottleneck — especially on HDDs or network-mounted drives — reducing overall performance.

Recommendations: - Use a moderate number of threads (e.g., 2–4) when writing results.
- Prefer SSDs or local storage for parallel jobs.
- Avoid excessive simultaneous writes from multiple processes.

Comparing R and C++ parallelism

Aspect R API (bdblockMult) C++ API (bdblockmult_hdf5_rcpp)
Execution layer R (multithread) Native C++ (OpenMP)
Parameter threads threads
Typical speedup 2–4× (depends on I/O) 3–8× (depends on CPU and disk)
Memory usage Higher (in-memory objects) Lower (block streaming)
Scalability Limited by R overhead Limited by I/O bandwidth
Ease of use Very easy Requires C++ integration

Practical considerations

TipBest practices
  • Use threads equal to the number of physical cores, not logical threads.
  • Avoid extreme values on shared systems (leave 1–2 cores free).
  • Balance threads and block_size for your memory and disk speed.
  • For iterative analyses, test different setups and measure performance.
NoteParallel overhead

Parallelism is not always faster for small datasets.
If data fit easily in memory, the cost of splitting, scheduling, and combining tasks can exceed the performance gain.

Example: HDF5 matrix multiplication

The same principle applies to on-disk computations.

# Example using on-disk data
set.seed(1)
A <- matrix(rnorm(1e4), nrow = 100, ncol = 100)
B <- matrix(rnorm(1e4), nrow = 100, ncol = 100)

# Create HDF5 files
bdCreate_hdf5_matrix("par_demo.h5", A, group = "demo", dataset = "A",
                     overwriteFile = TRUE, overwriteDataset = TRUE)
$fn
[1] "par_demo.h5"

$ds
[1] "demo/A"
bdCreate_hdf5_matrix("par_demo.h5", B, group = "demo", dataset = "B",
                     overwriteFile = FALSE, overwriteDataset = TRUE)
$fn
[1] "par_demo.h5"

$ds
[1] "demo/B"
# Run block-wise multiplication using multiple threads
system.time(
  bdblockmult_hdf5("par_demo.h5",
                   group = "demo", A = "A",
                   groupB = "demo", B = "B",
                   block_size = 128, threads = 4)
)
   user  system elapsed 
  0.003   0.001   0.004

This function automatically distributes block computations across available threads while reading and writing data directly from HDF5.

Summary

  • Parallelism in BigDataStatMeth is achieved via block-wise computation using the threads parameter.
  • The R API offers an easy entry point, while the C++ API maximizes performance.
  • Optimal performance requires balancing threads, block size, and I/O throughput.
  • Be aware of potential I/O bottlenecks during parallel writes.
  • Experiment with different configurations to find the best trade-off for your system.
