HDF5 Operations

Why operations matter

Once your data lives in an HDF5 file, BigDataStatMeth can operate on it without loading everything into memory. This enables efficient pipelines for writing, reading/inspecting, and computing on large matrices using block-wise algorithms.

Important

Key idea — Operate on-disk when data does not fit in RAM, and in-memory when it does. The API lets you choose the right tool for each situation.

Writing data

Create a file, a group, and a dataset in one step. If any of them do not exist, they are created automatically.

  • R
  • C++ (not executed)
library(BigDataStatMeth)
library(rhdf5)

# Example matrix
X <- matrix(rnorm(12), nrow = 3)

# Write to HDF5: creates file, group and dataset if missing
bdCreate_hdf5_matrix(
  filename = "ops_demo.h5",
  object   = X,
  group    = "myData",
  dataset  = "X",
  overwriteFile = TRUE
)
$fn
[1] "ops_demo.h5"

$ds
[1] "myData/X"
# Inspect structure
rhdf5::h5ls("ops_demo.h5")
    group   name       otype dclass   dim
0       / myData   H5I_GROUP             
1 /myData      X H5I_DATASET  FLOAT 3 x 4

Tip

Good practice — Use descriptive names for groups/datasets (e.g., /raw, /processed, /results) to keep analyses clear and reproducible.

Reading data (inspect with rhdf5)

BigDataStatMeth reads HDF5 internally in C++ during computations. For manual inspection from R, use the rhdf5 package.

# List objects
rhdf5::h5ls("ops_demo.h5")
    group   name       otype dclass   dim
0       / myData   H5I_GROUP             
1 /myData      X H5I_DATASET  FLOAT 3 x 4
# Read a (sub)matrix if needed, e.g. first 2 rows, first 3 cols
rhdf5::h5read("ops_demo.h5", name = "myData/X",
              index = list(1:2, 1:3))
           [,1]      [,2]      [,3]
[1,] -0.1373560 0.5608898  1.092973
[2,]  0.6310405 0.1134788 -1.717751
Note

Note — There is no general read_* wrapper in BigDataStatMeth because computational functions access HDF5 directly in C++. For exploration and debugging, use rhdf5::h5ls() and rhdf5::h5read().

Operating by blocks (on-disk vs in-memory)

Many functions in BigDataStatMeth use block-wise computation to work directly from disk.

  • R
  • C++ (not executed)
library(BigDataStatMeth)

# In-memory block multiplication (data in RAM) — make dimensions conformable
set.seed(1)
A <- matrix(rnorm(2000), nrow = 100, ncol = 20)  # 100x20
B <- matrix(rnorm(600),  nrow = 20,  ncol = 30)  # 20x30

# Reference result in memory (baseline)
AB_inmem <- A %*% B

# Compare with BigDataStatMeth in-memory block function
AB_bdmem <- bdblockMult(A, B)

# Store A and B in HDF5 for on-disk operation
bdCreate_hdf5_matrix("ops_demo.h5", A, group = "ops", dataset = "A",
                     overwriteFile = FALSE, overwriteDataset = TRUE)
$fn
[1] "ops_demo.h5"

$ds
[1] "ops/A"
bdCreate_hdf5_matrix("ops_demo.h5", B, group = "ops", dataset = "B",
                     overwriteFile = FALSE, overwriteDataset = TRUE)
$fn
[1] "ops_demo.h5"

$ds
[1] "ops/B"
# On-disk block multiplication (HDF5)
# (Result may be written to disk or returned as an R object depending on the implementation)
res_hdf5 <- bdblockmult_hdf5("ops_demo.h5",
                             group = "ops", A = "A",
                             groupB = "ops", B = "B")

# Quick check (if res_hdf5 returns an R matrix)
if (exists("res_hdf5") && is.matrix(res_hdf5)) {
  cat("Max abs diff (RAM vs HDF5):", max(abs(AB_inmem - res_hdf5)), "\n")
}
/** C++
------------------------------------------------------------
Block-wise HDF5 multiplication using BigDataStatMeth API
Adapts the pattern you provided (datasets A, B -> C),
including open, create, multiply, and robust exception handling.
**/

#include "BigDataStatMeth.h"

using namespace BigDataStatMeth;

int main() {
  const std::string filename      = "ops_demo.h5";
  const std::string strsubgroupIn = "ops";
  const std::string strsubgroupInB= "ops";
  const std::string A             = "A";
  const std::string B             = "B";
  const std::string strsubgroupOut= "ops";
  const std::string strdatasetOut = "AB";
  const bool bforce   = true;   // overwrite output dataset if exists

  // multiplication options
  const bool btransA  = false;
  const bool btransB  = false;
  const bool paral    = true;    // enable parallel if available
  const int  block_size = 1024;  // tune per system
  const int  threads    = 4;     // tune per system

  hdf5Dataset* dsA = nullptr;
  hdf5Dataset* dsB = nullptr;
  hdf5Dataset* dsC = nullptr;

  try {
    H5::Exception::dontPrint();

    dsA = new hdf5Dataset(filename, strsubgroupIn, A, false);
    dsA->openDataset();

    dsB = new hdf5Dataset(filename, strsubgroupInB, B, false);
    dsB->openDataset();

    if (dsA->getDatasetptr() != nullptr && dsB->getDatasetptr() != nullptr) {
      dsC = new hdf5Dataset(filename, strsubgroupOut, strdatasetOut, bforce);
      BigDataStatMeth::multiplication(dsA, dsB, dsC, btransA, btransB, paral, block_size, threads);
      delete dsC; dsC = nullptr;
    }

    delete dsA; dsA = nullptr;
    delete dsB; dsB = nullptr;

    return 0;
  } catch (H5::FileIException& error) {
    BigDataStatMeth::checkClose_file(dsA, dsB, dsC);
    Rcpp::Rcerr << "\nC++ exception blockmult_hdf5 (File IException)\n";
  } catch (H5::DataSetIException& error) {
    BigDataStatMeth::checkClose_file(dsA, dsB, dsC);
    Rcpp::Rcerr << "\nC++ exception blockmult_hdf5 (DataSet IException)\n";
  } catch (std::exception& ex) {
    BigDataStatMeth::checkClose_file(dsA, dsB, dsC);
    Rcpp::Rcerr << "C++ exception blockmult_hdf5: " << ex.what();
  } catch (...) {
    BigDataStatMeth::checkClose_file(dsA, dsB, dsC);
    Rcpp::Rcerr << "C++ exception blockmult_hdf5 (unknown reason)";
  }
  return 1;
}
Tip

Tip — Block-wise operations reduce memory pressure and allow you to scale to very large matrices. They may be slower than RAM-based computations because of disk I/O, but they enable analyses that would otherwise be impossible.

Mini benchmark: RAM vs HDF5

We compare three approaches for multiplying matrices: base R (%*%), bdblockMult() (in-memory), and bdblockmult_hdf5() (HDF5).

library(microbenchmark)
set.seed(1)

A <- matrix(rnorm(40000), nrow = 200, ncol = 200)  # 200x200
B <- matrix(rnorm(40000), nrow = 200, ncol = 200)  # 200x200

# Prepare HDF5 datasets (overwrite datasets if they exist)
bdCreate_hdf5_matrix("ops_bench.h5", A, group = "bench", dataset = "A",
                     overwriteFile = TRUE, overwriteDataset = TRUE)
$fn
[1] "ops_bench.h5"

$ds
[1] "bench/A"
bdCreate_hdf5_matrix("ops_bench.h5", B, group = "bench", dataset = "B",
                     overwriteFile = FALSE, overwriteDataset = TRUE)
$fn
[1] "ops_bench.h5"

$ds
[1] "bench/B"
bm_ops <- microbenchmark(
  base_R   = A %*% B,
  bd_mem   = bdblockMult(A, B),
  bd_hdf5  = bdblockmult_hdf5("ops_bench.h5",
                              group = "bench", A = "A",
                              groupB = "bench", B = "B"),
  times = 5
)

bm_ops
Unit: microseconds
    expr      min       lq     mean   median       uq       max neval
  base_R 2919.364 2924.120 2968.572 2942.734 3003.988  3052.655     5
  bd_mem  349.402  599.174 1587.955  718.689 1371.983  4900.525     5
 bd_hdf5 2079.766 2150.901 4093.005 2326.668 2359.099 11548.593     5
# Robust aggregation
s  <- as.data.frame(bm_ops)
ag <- aggregate(time ~ expr, data = s, FUN = median, na.rm = TRUE)
ag$ms <- ag$time / 1e6
ag <- ag[order(ag$expr), ]
med_ops <- setNames(ag$ms, ag$expr)
rel_ops <- med_ops / med_ops["bd_mem"]

Median times (ms):
- bdblockMult() (RAM): 0.719
- base R %*%: 2.943
- bdblockmult_hdf5() (HDF5): 2.327

Compared to bdblockMult() (in-memory), HDF5 is 3.237 × slower and base R is 4.095 × slower/faster (depending on your machine). This difference is expected due to disk I/O, and the benefit is that HDF5 scales to matrices much larger than RAM.

Data flow (visual)

This diagram summarizes the flow for block-wise operations: read blocks from disk → compute → write results back to disk.
We iterate over blocks: read a block from /bench/A and /bench/B, compute in RAM, and write the corresponding block to /bench/AB. Repeat until all blocks are processed. No full matrix is loaded in memory.

Good practices

  • Choose descriptive names for groups/datasets.
  • When data is large, operate by blocks and avoid full loads to RAM.
  • Benchmark a few block sizes on your system to find a good tradeoff.
  • Keep data, metadata and results in one HDF5 to improve traceability.
  • Use rhdf5 for inspection; rely on BigDataStatMeth for computation.
TipSummary
  • Write once: create files, groups, and datasets automatically.
  • Inspect with rhdf5, compute with BigDataStatMeth.
  • Use in-memory operations for speed on small/medium data.
  • Switch to HDF5 block-wise operations to scale beyond RAM.
