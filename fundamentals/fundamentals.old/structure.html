Quick Overview

NoteðŸ“‹ TL;DR - HDF5 Structure Basics

HDF5 files contain two main object types:

  1. Groups: Containers (like folders) that organize content
  2. Datasets: Arrays of data (matrices, vectors, scalars)

Key concepts: - Hierarchical organization: Groups can contain groups and datasets - Paths: Objects identified by Unix-like paths (/group/dataset) - Attributes: Metadata attached to any object - Root group: Every file has a root group (/)

Think of it as: - Groups = Folders - Datasets = Files - Attributes = File properties - File = Drive with complete file system

Datasets: The Data

Datasets are multidimensional arrays that store your actual data.

Key characteristics:

  • Homogeneous: All elements same data type
  • Dimensionality: 0D (scalar) to nD arrays
  • Size: Can be enormous (petabytes)
  • Chunked: Stored in optimal blocks for access

Paths and Navigation

HDF5 uses Unix-style paths to identify objects:

Absolute paths (from root):

/group/subgroup/dataset
/data/raw/experiment1

Relative paths (from current location):

../other_group/dataset

Special paths:

/           Root group
.           Current group
..          Parent group

Technical Details

TipðŸ”¢ Technical Specifications (click to expand)

Group Structure

Groups are implemented as B-trees for efficient lookup:

Group properties: - Creation order: Tracked if requested - Link storage: Dense or compact - B-tree order: Typically 16 - Overhead: ~1 KB per group

Group types:

  1. Compact: For small groups (< 8 links)
  2. Dense: For medium groups
  3. B-tree: For large groups (> 1000 links)

Dataset Storage

Datasets can use three layouts:

1. Contiguous Layout

[Header | Data in one continuous block]
  • Simple, fast for sequential access
  • Cannot be extended
  • No compression
  • Best for: Small, static datasets

2. Chunked Layout

[Header | Chunk1 | Chunk2 | ... | ChunkN]
  • Enables compression
  • Extensible
  • Partial I/O
  • Best for: Large datasets

3. Compact Layout

[Header with embedded data]
  • For tiny datasets (< 64 KB)
  • Minimal overhead
  • Best for: Metadata-like data

Datatype System

HDF5 has a rich type system:

Atomic types:

H5T_NATIVE_INT       Integer (32-bit)
H5T_NATIVE_LONG      Long integer (64-bit)
H5T_NATIVE_FLOAT     Float (32-bit)
H5T_NATIVE_DOUBLE    Double (64-bit)
H5T_STRING           String (variable/fixed)

Compound types:

typedef struct {
    int    sample_id;
    double expression;
    char   tissue[20];
} SampleData;

Array types:

Array of [3 Ã— 3] matrices

Enumeration types:

enum Status {CONTROL=0, CASE=1, UNKNOWN=2}

Link Types

HDF5 supports three link types:

1. Hard Links

/data/original  -> Dataset
/data/copy      -> Same dataset (different path)
  • Multiple paths to same object
  • Object deleted when all links removed
  • Like Unix hard links

2. Soft Links (Symbolic Links)

/data/link -> "/other/dataset"
  • Path stored as string
  • Can be broken (dangling)
  • Like Unix symlinks

3. External Links

/local/link -> "other_file.hdf5:/path/to/dataset"
  • Links to object in different file
  • Enables distributed datasets
  • Files opened on-demand

Attribute Limitations

Attributes have specific constraints:

  • Size limit: Typically 64 KB (configurable)
  • No chunking: Always stored contiguously
  • No compression: Limited by size
  • Use case: Small metadata only

Guidelines: - Small values: Use attributes - Large arrays: Use separate datasets - Descriptive info: Use attributes - Analysis results: Use datasets

Object References

HDF5 supports references (pointers) to objects:

Region references:

# Reference to specific region of dataset
ref <- H5Rcreate(file, "/data/matrix", 
                 H5S_SELECT_HYPERSLABS,
                 start=c(1,1), count=c(100,100))

Object references:

# Reference to entire dataset
ref <- H5Rcreate(file, "/data/matrix", H5S_SELECT_ALL)

Use cases: - Creating indices - Linking related datasets - Building complex structures

Working with Structure

Creating Groups and Datasets

Using rhdf5

library(rhdf5)

# Create file
h5createFile("analysis.hdf5")

# Create group hierarchy
h5createGroup("analysis.hdf5", "data")
h5createGroup("analysis.hdf5", "data/raw")
h5createGroup("analysis.hdf5", "data/processed")

# Create dataset in group
h5createDataset("analysis.hdf5", 
                "data/raw/genotypes",
                dims = c(1000, 5000),
                storage.mode = "double",
                chunk = c(100, 500),
                level = 6)

# Write data
data <- matrix(rnorm(5000000), 1000, 5000)
h5write(data, "analysis.hdf5", "data/raw/genotypes")

Using BigDataStatMeth

library(BigDataStatMeth)

# Create entire structure in one call
data <- matrix(rnorm(5000000), 1000, 5000)

bdCreate_hdf5_matrix(
  filename = "analysis.hdf5",
  object = data,
  group = "data/raw",      # Creates all parent groups
  dataset = "genotypes",
  overwriteFile = TRUE
)

# Automatically creates:
# /data/
# /data/raw/
# /data/raw/genotypes

Working with Attributes

Add attributes

# Using rhdf5
h5writeAttribute("Genotype data", 
                 "analysis.hdf5", 
                 "data/raw/genotypes",
                 name = "description")

h5writeAttribute(1000, 
                 "analysis.hdf5", 
                 "data/raw/genotypes",
                 name = "n_samples")

h5writeAttribute(5000, 
                 "analysis.hdf5", 
                 "data/raw/genotypes",
                 name = "n_snps")

# Add multiple attributes at once
attrs <- list(
  description = "Quality-controlled SNP data",
  maf_threshold = 0.05,
  missingness_threshold = 0.1,
  date_processed = Sys.Date()
)

for (name in names(attrs)) {
  h5writeAttribute(attrs[[name]], 
                   "analysis.hdf5",
                   "data/processed/qc_filtered",
                   name = name)
}

Read attributes

Organizing a Complete Analysis

Example: End-to-end workflow structure

library(BigDataStatMeth)

# 1. Create base structure
filename <- "genomic_analysis.hdf5"

# 2. Add raw data

  filename = filename,
  object = raw_genotypes,
  group = "data/raw",
  dataset = "genotypes",
  overwriteFile = TRUE
)

  filename = filename,
  object = phenotypes,
  group = "data/raw",
  dataset = "phenotypes",
  overwriteFile = FALSE
)

# 3. QC and save to processed
filtered <- bdRemoveMAF_hdf5(
  filename = filename,
  group = "data/raw",
  dataset = "genotypes",
  outgroup = "data/processed",
  outdataset = "qc_filtered",
  maf = 0.05
)

# 4. Perform PCA
pca_result <- bdPCA_hdf5(

Naming Conventions

Recommended conventions:

# Groups: lowercase, underscores
/data/raw_data/experiment_01/

# Datasets: lowercase, descriptive
/data/raw/genotype_matrix
/data/processed/normalized_expression

# Attributes: camelCase or snake_case
description
processingDate
n_samples

# Versions: suffix with date or version
/analysis/pca_20240115/
/analysis/pca_v2/
