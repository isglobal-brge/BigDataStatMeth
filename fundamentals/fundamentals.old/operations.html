Quick Overview

Noteüìã TL;DR - HDF5 Operations

Basic operations on HDF5 data:

CRUD Operations: - Create: Create files, groups, and datasets - Read: Access entire datasets or subsets - Update: Modify existing data - Delete: Remove groups or datasets

BigDataStatMeth provides: - High-level statistical operations on HDF5 - Matrix algebra directly on disk - Block-wise processing for large data - Both R and C++ APIs

When to use what: - BigDataStatMeth functions: Statistical analysis, matrix operations - rhdf5 (when needed): File inspection (h5ls), reading results (h5read)

Conceptual Understanding

The CRUD Paradigm

All data operations follow the standard CRUD pattern:

Operation Purpose BigDataStatMeth Approach
Create New data bdCreate_hdf5_matrix()
Read Access data Automatic in operations
Update Modify data Result of operations
Delete Remove data bdRemove_hdf5_element()

BigDataStatMeth Philosophy

BigDataStatMeth is designed for statistical methods and matrix algebra on HDF5 data, not general file I/O:

Focus areas: 1. Matrix operations: Multiplication, crossproduct, etc. 2. Statistical methods: PCA, SVD, regression 3. Data processing: Normalization, QC, imputation 4. Performance: Block-wise, parallel processing

Not focused on: - Low-level file inspection - Generic data reading/writing - File system operations

For those tasks, use rhdf5 package as complement.

Reading Strategies

BigDataStatMeth implements efficient reading internally:

1. Block-wise Reading (Large datasets)

Dataset [1,000,000 √ó 10,000]
    ‚Üì
Read blocks of 10,000 rows
    ‚Üì
Process each block
    ‚Üì
Aggregate results

Used automatically by functions like bdPCA_hdf5, bdblockmult_hdf5.

2. Partial Access (Subsets)

Dataset [10,000 √ó 10,000]
    ‚Üì
Read only needed regions
    ‚Üì
Process subset

Used by functions like bdsubset_hdf5_dataset.

3. Parallel Access (Multiple workers)

Dataset divided into blocks
    ‚Üì
Multiple threads read simultaneously
    ‚Üì
Parallel processing

Enabled with paral=TRUE, threads=n in functions.

Technical Details

Tipüî¢ Technical Specifications (click to expand)

File Access Patterns

BigDataStatMeth operations use optimized access patterns:

Sequential block access:

// Efficient: Read contiguous blocks
for (block in blocks) {
  data = read_hdf5_block(file, dataset, block)
  process(data)
}

Random access optimization:

// Cached: Frequently accessed chunks
chunk_cache_size = 10 MB
recently_accessed = cached_automatically

Block Processing

All BigDataStatMeth functions use block-wise algorithms:

Block size selection:

# Automatic (recommended)
bdPCA_hdf5(file, group, dataset)
# Uses optimal block size based on:
# - Available memory
# - Dataset dimensions
# - Cache efficiency

# Manual (advanced)
bdblockmult_hdf5(file, group, A, B, 
                 blocksize = 256)

Block alignment: Operations align blocks with HDF5 chunks for efficiency:

HDF5 chunks: [1000 √ó 1000]
Block size:  [1000 √ó 1000]  ‚Üê Aligned (fast)
Block size:  [500 √ó 500]    ‚Üê Suboptimal (slower)

Operation Types

In-core operations: - Small datasets fit in memory - Processed directly - Fastest performance

Out-of-core operations: - Large datasets on disk - Block-wise processing - Scalable to any size

Hybrid operations: - Critical data in memory - Bulk data on disk - Optimized I/O

Parallel I/O

BigDataStatMeth supports parallel operations:

Thread-level parallelism:

# Enable parallel processing
bdPCA_hdf5(file, group, dataset,
           paral = TRUE,
           threads = 4)

Process-level parallelism: - Multiple R processes can read simultaneously - Safe for read-only operations - Coordination needed for writes

BigDataStatMeth Operations

Available Operation Categories

BigDataStatMeth provides comprehensive HDF5 operations organized by purpose:

1. Data Creation and Import

Create HDF5 datasets from R objects:

library(BigDataStatMeth)

# Create HDF5 matrix from R object
data <- matrix(rnorm(10000), 100, 100)

bdCreate_hdf5_matrix(
  filename = "data.hdf5",
  object = data,
  group = "analysis",
  dataset = "matrix_a"
)

Related functions: - bdCreate_hdf5_matrix() - Create dataset from matrix - bdCreate_hdf5_group() - Create groups - bdImportData_hdf5() - Import from URLs/files - bdImportTextFile_hdf5() - Import text files

See also: HDF5 Utilities (when categorized)

2. Matrix Operations

Perform linear algebra directly on HDF5:

# Matrix multiplication
result <- bdblockmult_hdf5(
  filename = "data.hdf5",
  group = "analysis",
  A = "matrix_a",
  B = "matrix_b",
  blocksize = 128,
  paral = TRUE
)

# Crossproduct: t(A) %*% A
cp <- bdCrossprod_hdf5(
  filename = "data.hdf5",
  group = "analysis",
  A = "matrix_a"
)

# Transposed crossproduct: A %*% t(A)
tcp <- bdtCrossprod_hdf5(
  filename = "data.hdf5",
  group = "analysis",
  A = "matrix_a"
)

Available operations: - Multiplication: bdblockmult_hdf5(), bdblockmult_sparse_hdf5() - Crossproduct: bdCrossprod_hdf5(), bdtCrossprod_hdf5() - Addition/Subtraction: bdblockSum_hdf5(), bdblockSubstract_hdf5() - Decompositions: bdCholesky_hdf5(), bdQR_hdf5()

See also: - Linear Algebra functions - C++ Matrix operations

3. Statistical Methods

Apply statistical methods on large-scale data:

# Principal Component Analysis
pca_result <- bdPCA_hdf5(
  filename = "data.hdf5",
  group = "analysis",
  dataset = "matrix_a",
  k = 20,              # Number of components
  bcenter = TRUE,      # Center data
  bscale = FALSE,      # Don't scale
  threads = 4          # Parallel processing
)

# Results automatically stored in HDF5:
# /PCA/matrix_a/components
# /PCA/matrix_a/eigenvalues
# /PCA/matrix_a/variance

Available methods: - PCA: bdPCA_hdf5() - SVD: bdSVD_hdf5() - Eigenvalues: bdEigen_hdf5() - Correlation: bdCorr_hdf5() - Normalization: bdNormalize_hdf5()

See also: Statistical Methods

4. Data Processing and QC

Process and clean genomic/omic data:

# Remove SNPs with high missing data
filtered <- bdRemovelowdata_hdf5(
  filename = "genomics.hdf5",
  group = "raw",
  dataset = "genotypes",
  outgroup = "processed",
  outdataset = "qc_filtered",
  bycols = TRUE,
  pcent = 0.05  # 5% threshold
)

# Filter by Minor Allele Frequency
maf_filtered <- bdRemoveMAF_hdf5(
  filename = "genomics.hdf5",
  group = "processed",
  dataset = "qc_filtered",
  outgroup = "processed",
  outdataset = "maf_filtered",
  maf = 0.05
)

# Impute missing SNPs
bdImputeSNPs_hdf5(
  filename = "genomics.hdf5",
  group = "processed",
  dataset = "maf_filtered",
  bycols = TRUE
)

Available utilities: - bdRemovelowdata_hdf5() - Remove high-missing data - bdRemoveMAF_hdf5() - MAF filtering - bdImputeSNPs_hdf5() - SNP imputation - bdNormalize_hdf5() - Data normalization

5. Dataset Manipulation

Organize and manage datasets:

# Get list of datasets in group
datasets <- bdgetDatasetsList_hdf5(
  filename = "data.hdf5",
  group = "analysis"
)

# Get dataset dimensions
dims <- bdgetDim_hdf5(
  filename = "data.hdf5",
  group = "analysis",
  dataset = "matrix_a"
)

# Subset dataset
bdsubset_hdf5_dataset(
  filename = "data.hdf5",
  group = "analysis",
  dataset = "matrix_a",
  rows = 1:100,
  cols = 1:50,
  outgroup = "subsets",
  outdataset = "small_matrix"
)

# Bind datasets (rbind/cbind)
bdBind_hdf5_datasets(
  filename = "data.hdf5",
  group = "parts",
  datasets = c("part1", "part2", "part3"),
  outgroup = "combined",
  outdataset = "full_matrix",
  by_rows = TRUE
)

Available functions: - bdgetDatasetsList_hdf5() - List datasets - bdgetDim_hdf5() - Get dimensions - bdsubset_hdf5_dataset() - Extract subsets - bdBind_hdf5_datasets() - Combine datasets - bdSplit_matrix_hdf5() - Split datasets - bdSort_hdf5_dataset() - Sort datasets

C++ API for Advanced Users

For method developers and advanced users, BigDataStatMeth provides a complete C++ API:

Core HDF5 Classes

#include <BigDataStatMeth.hpp>

// File management
BigDataStatMeth::hdf5File file("data.hdf5", "r");

// Dataset access
BigDataStatMeth::hdf5Dataset dataset = file.getDataset("group/dataset");

// Read data
Eigen::MatrixXd data = dataset.read();

// Write data
dataset.write(result);

Available classes: - hdf5File - File operations - hdf5Dataset - Dataset access - hdf5Group - Group management - hdf5DatasetInternal - Internal operations - hdf5DiagonalMatrix - Specialized matrix - hdf5Dims - Dimension handling

See also: - C++ Classes documentation - C++ Functions documentation

When to Use rhdf5

While BigDataStatMeth handles most operations, rhdf5 is useful for:

File inspection:

library(rhdf5)

# View file structure
h5ls("data.hdf5")

# Check what's in a group
h5ls("data.hdf5", recursive = FALSE)

Reading results:

# After BigDataStatMeth operation
pca_result <- bdPCA_hdf5(...)

# Read components for plotting
library(rhdf5)
components <- h5read("data.hdf5", "/PCA/dataset/components")
eigenvalues <- h5read("data.hdf5", "/PCA/dataset/eigenvalues")

# Plot
plot(components[, 1], components[, 2])

# Always close
h5closeAll()

Low-level operations not in BigDataStatMeth:

# Add attributes
h5writeAttribute(metadata, "data.hdf5", 
                 "group/dataset", name = "description")

# Read attributes
attrs <- h5readAttributes("data.hdf5", "group/dataset")

Complete Workflow Example

Putting it all together:

library(BigDataStatMeth)
library(rhdf5)

# 1. Create HDF5 file from R data
data <- matrix(rnorm(100000), 1000, 100)

bdCreate_hdf5_matrix(
  filename = "analysis.hdf5",
  object = data,
  group = "raw",
  dataset = "data"
)

# 2. Normalize data
bdNormalize_hdf5(
  filename = "analysis.hdf5",
  group = "raw",
  dataset = "data",
  outgroup = "processed",
  outdataset = "normalized",
  bcenter = TRUE,
  bscale = TRUE
)

# 3. Compute correlation matrix
bdCorr_hdf5(
  filename = "analysis.hdf5",
  group = "processed",
  dataset = "normalized"
)

# 4. Perform PCA
pca <- bdPCA_hdf5(
  filename = "analysis.hdf5",
  group = "processed",
  dataset = "normalized",
  k = 10,
  threads = 4
)

# 5. Inspect results with rhdf5
h5ls("analysis.hdf5")

# 6. Read components for visualization
components <- h5read("analysis.hdf5", 
                     "/PCA/normalized/components")

# 7. Plot
plot(components[, 1], components[, 2],
     main = "First two principal components",
     xlab = "PC1", ylab = "PC2")

# 8. Clean up
h5closeAll()

Operation categories: 1. Data creation and import 2. Matrix operations (linear algebra) 3. Statistical methods (PCA, SVD, etc.) 4. Data processing and QC 5. Dataset manipulation
