Working with HDF5

Introduction

HDF5 (Hierarchical Data Format 5) is a file format designed for storing and managing large amounts of data. BigDataStatMeth provides native support for HDF5, allowing you to work with datasets larger than available RAM.

Why HDF5?

  • Efficient storage: Compressed data storage
  • Partial reading: Access subsets without loading entire dataset
  • Multiple datasets: Store multiple arrays in a single file
  • Metadata support: Attach descriptive information to datasets
  • Cross-platform: Works on all major operating systems

Creating HDF5 Files

Basic File Creation

library(BigDataStatMeth)

# Create a matrix
data <- matrix(rnorm(100000), nrow=1000, ncol=100)

# Save to HDF5
bdCreate_hdf5_matrix_file(
  filename = "mydata.h5",
  dataset = "mymatrix",
  data = data,
  chunk = c(100, 100)  # Chunk size for optimal access
)

Multiple Datasets

# Add another dataset to existing file
bdAdd_hdf5_matrix(
  filename = "mydata.h5",
  dataset = "another_matrix",
  data = matrix(rnorm(50000), nrow=500, ncol=100)
)

Reading HDF5 Data

Complete Dataset

# Read entire dataset
data <- bdRead_hdf5_matrix("mydata.h5", "mymatrix")

Partial Reading (Slicing)

# Read specific rows and columns
subset <- bdRead_hdf5_matrix(
  filename = "mydata.h5",
  dataset = "mymatrix",
  start_row = 1,
  start_col = 1,
  nrows = 100,
  ncols = 50
)

HDF5-Based Operations

Direct Operations

Perform computations without loading data into memory:

# Matrix multiplication using HDF5
result <- bdblockmult_hdf5(
  file1 = "mydata.h5",
  dataset1 = "mymatrix",
  file2 = "mydata.h5", 
  dataset2 = "mymatrix",
  blocksize = 128,
  paral = TRUE,
  output_file = "result.h5",
  output_dataset = "multiplication_result"
)

Normalization

# Center and scale data
bdNormalize_hdf5(
  filename = "mydata.h5",
  dataset = "mymatrix",
  output_dataset = "normalized_matrix",
  center = TRUE,
  scale = TRUE
)

Best Practices

1. Choose Appropriate Chunk Size

# For row-wise access
chunk <- c(100, ncol(data))  # Full width chunks

# For column-wise access  
chunk <- c(nrow(data), 10)  # Few columns per chunk

# For block operations
chunk <- c(100, 100)  # Square chunks

2. Use Compression

# Enable compression when creating files
bdCreate_hdf5_matrix_file(
  filename = "compressed.h5",
  dataset = "data",
  data = large_matrix,
  compression = 6  # Compression level 0-9
)

3. Clean Up Temporary Files

# Remove temporary datasets
bdRemove_hdf5_element("mydata.h5", "temporary_dataset")

# Or delete entire file when done
unlink("temporary.h5")

Advanced Topics

Working with Groups

# Create organized structure
bdCreateGroup_hdf5("mydata.h5", "/processed")
bdCreateGroup_hdf5("mydata.h5", "/raw")

# Add datasets to groups
bdAdd_hdf5_matrix(
  filename = "mydata.h5",
  dataset = "/processed/normalized",
  data = normalized_data
)

Batch Processing

# Process multiple files
files <- list.files(pattern = "\\.h5$")

results <- lapply(files, function(f) {
  bdPCA_hdf5(f, "dataset", k=10)
})

Troubleshooting

Common Issues

File locked errors: Ensure files are properly closed

# Always close HDF5 connections
rhdf5::H5close()

Memory issues: Use smaller block sizes

# Reduce blocksize if memory errors occur
result <- bdblockmult_hdf5(..., blocksize=64)

Slow performance: Check chunk alignment

# Verify chunk size matches access pattern
bdCheck_hdf5_chunks("mydata.h5", "mymatrix")
