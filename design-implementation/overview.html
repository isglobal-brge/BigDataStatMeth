Overview

Introduction

BigDataStatMeth is designed to bring scalability and efficiency to statistical computing in R by combining the expressiveness of R with the performance of C++ and the persistence of HDF5.

Its architecture is based on three complementary layers:

  • R Interface: user-friendly functions (bdblockMult, bdSVD_hdf5, …).
  • Rcpp Bridge — implemented using the open-source Rcpp package (Eddelbuettel and François, 2011), which provides the seamless interface between R and C++.
  • C++ Core Engine: performs intensive computations efficiently.
  • HDF5 File System: manages storage and I/O of large datasets.

This layered design ensures that computations scale smoothly from in-memory operations to on-disk processing of terabyte-scale matrices.

Design philosophy

The implementation of BigDataStatMeth follows three main principles:

Tip1. Modularity

Each functionality (I/O, linear algebra, decomposition, etc.) is encapsulated in dedicated modules and C++ classes. This makes the code maintainable and extendable.

Tip2. Block-wise computation

Operations are performed by blocks instead of loading entire matrices into memory. This design allows analyses that exceed RAM capacity and fits naturally with parallel processing.

Tip3. Interoperability

All data are stored in HDF5, a standard supported by R, Python, MATLAB, and C++. This guarantees reproducibility and seamless integration across platforms and programming languages.

Integration of R and C++

BigDataStatMeth uses the Rcpp interface to expose C++ functions to R. The R layer acts mainly as a controller, handling user inputs and parameter validation, while the heavy computations occur in C++.

  • Example R call
  • C++ counterpart (simplified)
result <- bdblockMult(A, B, threads = 4)
void bdblockMult_rcpp(const NumericMatrix& A,
                      const NumericMatrix& B,
                      int threads) {
    // Dispatch to parallel C++ multiplication kernel
    BigDataStatMeth::MatrixOps::blockMultiply(A, B, threads);
}

This separation ensures that R users interact with a simple interface, while developers and advanced users can extend the core behavior in C++.

Why block-wise algorithms

Unlike traditional R operations that load entire matrices into memory, BigDataStatMeth divides matrices into smaller blocks. Each block is processed individually, allowing: - computations that exceed system RAM,
- parallel execution across threads,
- reduced I/O overhead when combined with HDF5 chunking.

This approach makes BigDataStatMeth scalable by design.

Example: from R call to HDF5 output

A typical workflow combines all three layers:

  1. The user calls an R function such as bdSVD_hdf5().
  2. The R function passes parameters to the C++ backend via Rcpp.
  3. The C++ engine reads blocks of data from HDF5.
  4. Each block is processed (possibly in parallel).
  5. Results are written back to the same or a new HDF5 file.
Important

Key advantage: Each layer has a specific role, minimizing data movement between R and C++ and allowing computations on datasets larger than RAM.

Summary

  • BigDataStatMeth integrates R, C++ and HDF5 in a modular design.
  • Computations are executed by blocks, enabling scalability.
  • The Rcpp bridge connects user-level functions with optimized C++ routines.
  • The system achieves performance and interoperability without sacrificing usability.

Reference

Eddelbuettel, D. and François, R. (2011). Rcpp: Seamless R and C++ Integration. Journal of Statistical Software, 40(8), 1–18. https://doi.org/10.18637/jss.v040.i08
