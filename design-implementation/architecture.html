Internal Architecture

Purpose

This page explains how BigDataStatMeth is internally organized so that R users can call high-performance C++ code and work with HDF5 files without dealing with low-level details.

We will look at the layers (R → Rcpp → C++ → HDF5) and at the core C++ classes that actually exist in the package.

Layers in BigDataStatMeth

  • R layer: user-facing functions such as bdCreate_hdf5_matrix(), bdblockmult_hdf5(), …

  • Rcpp bridge: implemented through the Rcpp package (Eddelbuettel and François, 2011), which manages the interface between R and C++.

  • C++ modules: real work (read blocks, operate, write).

  • HDF5: persistent storage.

The goal is to move the heavy work to C++ but keep the R interface simple.

Computational modules

On top of these HDF5 classes, the package adds two families of operations:

  1. HDF5-based algebra
    headers in inst/include/hdf5Algebra/
    and sources in src/hdf5_*.cpp
    (for example: hdf5_blockmult.cpp, hdf5_blockCrossprod.cpp, hdf5_matrixQR.cpp).
    These work directly on data stored in HDF5 and use block-wise access.

  2. In-memory algebra
    headers in inst/include/memAlgebra/
    and sources in src/mem_*.cpp
    (for example: mem_blockmult.cpp, mem_crossprod.cpp).
    These work on R matrices already in memory.

So the architecture is not “a single MatrixOps class” but a set of HDF5 utilities plus algebra headers (hdf5Algebra, memAlgebra) and thin R wrappers.

R → C++ → HDF5: real flow

Let’s rewrite the flow of bdCreate_hdf5_matrix() in simplified form:

  1. R calls:

    bdCreate_hdf5_matrix(
  filename = "example.h5",
  object   = A,
  group    = "INPUT",
  dataset  = "A",
  overwriteFile    = TRUE,
  overwriteDataset = TRUE
)

  2. Rcpp forwards the arguments to C++. Communication between the R and C++ layers is implemented through the Rcpp interface (Eddelbuettel and François, 2011), which automatically handles data transfer and type conversion between R objects and their C++ equivalents.

  3. C++ does (simplified):

    • create or open hdf5File(filename, overwrite)
    • create/open group "INPUT" using hdf5Dataset constructor
    • create dataset "A" with detected data type
    • write the matrix (or data.frame converted to matrix)
    • write dimnames using hdf5Dims

  4. C++ cleans up objects in a try { ... } catch(...) { ... } block and reports errors to R via Rf_error(...).

Error handling

All C++ HDF5 code is wrapped in try/catch and errors are surfaced to R:

try {
    objFile = new BigDataStatMeth::hdf5File(filename, bforceFile);
    objFile->createFile();
    // ...
} catch (std::exception &ex) {
    Rf_error("c++ exception bdCreate_hdf5_matrix %s", ex.what());
}

This pattern is consistent across the HDF5 utilities and ensures R sees a regular error instead of a crash.

Summary

  • The package is organized in utilities (hdf5File, hdf5Group, hdf5Dataset, hdf5Dims) plus algebra modules (HDF5 and in-memory).
  • R only orchestrates the calls; the heavy work happens in C++.
  • The modular design allows adding new HDF5-based methods without changing the R interface.

Reference

Eddelbuettel, D. and François, R. (2011). Rcpp: Seamless R and C++ Integration. Journal of Statistical Software, 40(8), 1–18. https://doi.org/10.18637/jss.v040.i08
