ğŸš€ Quick Start

Prerequisites

  1. R packages:

    install.packages(c("roxygen2", "xml2", "stringr", "glue", "purrr", "quarto"))

  2. System tools:

    • Doxygen: brew install doxygen (macOS) or sudo apt-get install doxygen (Linux)
    • Graphviz: brew install graphviz (macOS) or sudo apt-get install graphviz (Linux)
    • Quarto CLI: Download from https://quarto.org/docs/get-started/

Initial Setup

  1. Create folder structure:

    cd BigDataStatMeth-docs/_scripts
Rscript setup_docs_structure.R

  2. Build documentation (prototype mode):

    cd BigDataStatMeth-docs/_scripts
Rscript build_docs.R

  3. Preview the site:

    cd BigDataStatMeth-docs/_scripts
Rscript build_docs.R preview

    Or directly with Quarto:

    cd BigDataStatMeth-docs
quarto preview

ğŸ“ Usage

Build Commands

From BigDataStatMeth-docs/_scripts/:

# Build prototype documentation (3 R functions + 1 C++ class)
Rscript build_docs.R

# Build full documentation (all functions)
Rscript build_docs.R full

# Clean generated files
Rscript build_docs.R clean

# Preview site in browser
Rscript build_docs.R preview

Prototype Mode

By default, the build script works in prototype mode and extracts:

R Functions (from package): - bdblockmult - bdPCA_hdf5 - bdCreate_hdf5_matrix_file

C++ Classes (from inst/include): - hdf5datasets

To extract all documentation, use full mode:

Rscript build_docs.R full

Manual Steps

If you want to run steps individually:

cd BigDataStatMeth-docs/_scripts

# Step 1: Generate Doxygen XML
doxygen Doxyfile

# Step 2: Extract Roxygen documentation
Rscript extract_roxygen.R

# Step 3: Convert Doxygen XML to Quarto
Rscript doxygen2quarto.R

# Step 4: Render Quarto site
cd ..
quarto render

ğŸ¨ Customization

Categorizing Functions

Functions are organized into categories. To classify a function, edit _scripts/extract_roxygen.R:

FUNCTION_CATEGORIES <- list(
  algebra = c("bdblockmult", "bdCrossprod", "bdtCrossprod"),
  statistics = c("bdPCA_hdf5", "bdSVD", "bdlm_paral"),
  hdf5 = c("bdCreate_hdf5_matrix_file", "bdAdd_hdf5_matrix", "bdRemovelowdata")
)

Note: Functions not listed in any category are automatically placed in the â€œOther Functionsâ€ category. You can reclassify them at any time by adding them to the appropriate category list.

ğŸ“Š Generated Content

The following directories are auto-generated and git-ignored:

  • api-reference/ - Function and class documentation
  • _site/ - Rendered website
  • _scripts/doxygen_xml/ - Intermediate Doxygen XML

Never edit files in these directories manually - they will be overwritten on next build.

ğŸ”„ Updating Documentation

When R functions change:

  1. Update Roxygen comments in package
  2. Rebuild docs: Rscript build_docs.R

When C++ code changes:

  1. Update Doxygen comments in headers
  2. Rebuild docs: Rscript build_docs.R

When guides/examples change:

  1. Edit .qmd files directly
  2. Render: quarto render or Rscript build_docs.R

ğŸŒ Deployment

GitHub Pages

  1. Build the site:

    Rscript build_docs.R full

  2. The site is in _site/

  3. Deploy to GitHub Pages:

    # Option 1: Using gh-pages branch
git checkout --orphan gh-pages
git add _site/*
git commit -m "Deploy documentation"
git push origin gh-pages

# Option 2: Using GitHub Actions
# (see .github/workflows/deploy-docs.yml example)

Other Hosting

The _site/ directory contains a static website that can be hosted anywhere: - Netlify: Drag and drop _site/ folder - AWS S3: Sync _site/ to bucket - Any web server: Copy _site/ contents
