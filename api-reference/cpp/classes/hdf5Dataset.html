5.1.4 inheritCompressionLevel()

void BigDataStatMeth::hdf5Dataset::inheritCompressionLevel(int level)

Inherit the compression level from an input dataset if not already set.

Parameters:

  • level (int): Compression level 0-9 read from an input dataset via getCompressionLevel()

Sets the compression level only when no explicit value has been assigned (i.e., compression_level < 0, the initial sentinel value). This allows header-level algorithms to propagate compression from input to output datasets while respecting any prior call to setCompressionLevel() made by a higher-level wrapper (R6 method, C++ API caller, or global option).

5.1.5 getCompressionLevel()

int BigDataStatMeth::hdf5Dataset::getCompressionLevel() const

Get the current compression level for this dataset instance.

Returns: Compression level 0-9

Returns the compression level that will be used by createDataset() when no explicit argument is passed. After openDataset() is called, this reflects the actual compression level stored in the HDF5 file (0 if the dataset was created without compression). Use this to propagate compression from an input dataset to output datasets: dsOut->setCompressionLevel(dsIn->getCompressionLevel());dsOut->createDataset(rows,cols,“numeric”); Compression level 0-9

5.1.6 createDataset()

virtual void BigDataStatMeth::hdf5Dataset::createDataset(BigDataStatMeth::hdf5Dataset *dsLike, std::string strdatatype, int compression_level=-1)

Create a dataset based on another dataset’s dimensions with optional compression.

Parameters:

  • dsLike (BigDataStatMeth::hdf5Dataset *): Reference dataset to copy dimensions from
  • strdatatype (std::string): Data type for the new dataset (“int”, “numeric”, “real”, “string”)
  • compression_level (int): Compression level (0=none, 1-9=gzip level, 6=balanced default) 0 = No compression 1-3 = Light compression (fast) 4-6 = Balanced compression (recommended) 7-9 = Maximum compression (slower)

Creates a new dataset with the same dimensions as the reference dataset but with a specified data type and compression settings. This is useful for creating derived datasets or transformations while maintaining dimensional consistency.

5.1.7 createUnlimitedDataset()

virtual void BigDataStatMeth::hdf5Dataset::createUnlimitedDataset(size_t rows, size_t cols, std::string strdatatype, int compression_level=-1)

Create an unlimited dataset with optional compression.

Parameters:

  • rows (size_t): Initial number of rows (used as chunk size)
  • cols (size_t): Initial number of columns (used as chunk size)
  • strdatatype (std::string): Data type for the dataset (“int”, “numeric”) Note: String type not supported for unlimited datasets
  • compression_level (int): Compression level (0=none, 1-9=gzip level, 6=balanced default) Recommended: 4-6 for unlimited datasets to balance compression ratio with extension performance

Creates a new HDF5 dataset with unlimited dimensions, allowing it to grow in size dynamically. The dataset is created with initial dimensions but can be extended using extendUnlimitedDataset(). Compression is fully compatible with unlimited datasets.

5.1.21 copyToFile()

virtual void BigDataStatMeth::hdf5Dataset::copyToFile(const std::string &dst_file, const std::string &dst_group, const std::string &dst_name, hsize_t blockrows=500, bool overwrite=true)

Copy this dataset to a dataset in a different (or same) HDF5 file.

Parameters:

  • dst_file (const std::string &): Path to the destination HDF5 file.
  • dst_group (const std::string &): Group path in the destination file.
  • dst_name (const std::string &): Dataset name in the destination file.
  • blockrows (hsize_t): Rows per copy block (default 500).
  • overwrite (bool): If true, overwrite existing destination dataset.

Reads the source block-by-block and writes to the destination, so memory usage is O(blockrows * ncols) regardless of matrix size. The destination file is created if it does not exist yet. If source and destination file are the same, prefer moveDataset() or a simple HDF5 group copy instead.
