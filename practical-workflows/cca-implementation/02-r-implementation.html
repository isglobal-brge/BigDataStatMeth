R Implementation

Overview

We’ll implement CCA by composing BigDataStatMeth functions. The key challenge: matrices too large for memory require block-wise processing.

Strategy: Build a helper function for block-wise QR, then use it in the main CCA algorithm.

Part 1: Block-wise QR Decomposition

Código Completo

getQRbyBlocks <- function(strdataset, file, m, center, scale, bcols, overwrt) {
    
    # Paso 1: Extraer grupo y nombre del dataset
    strgroup <- gsub("/.?$", "", strdataset)
    strdataset <- gsub("^.*/", "", strdataset)
    
    # Paso 2: Normalizar datos
    bdNormalize_hdf5(filename = file, group = strgroup, dataset = strdataset,
                     bcenter = center, bscale = scale, overwrite = overwrt)
    
    # Paso 3: Dividir en m bloques
    bdSplit_matrix_hdf5(filename = file,
                        group = paste0("NORMALIZED/", strgroup),
                        dataset = strdataset,
                        outgroup = paste0("Step1/", strdataset, "rows"),
                        nblocks = m, bycols = bcols, overwrite = overwrt)
    
    # Paso 4: Aplicar QR a cada bloque
    blocks <- bdgetDatasetsList_hdf5(file, paste0("Step1/", strdataset, "rows"))
    bdapply_Function_hdf5(filename = file,
                          group = paste0("Step1/", strdataset, "rows"),
                          datasets = blocks,
                          outgroup = paste0("Step2/", strdataset, "rows"),
                          func = "QR", overwrite = overwrt)
    
    # Paso 5: Apilar matrices R verticalmente
    blocks_qr <- bdgetDatasetsList_hdf5(file, paste0("Step2/", strdataset, "rows"))
    r_matrices <- blocks_qr[grep(".R$", blocks_qr)]
    bdBind_hdf5_datasets(filename = file,
                         group = paste0("Step2/", strdataset, "rows"),
                         datasets = r_matrices,
                         outgroup = "Step3/merged",
                         outdataset = paste0(strdataset, "Rt"),
                         func = "bindRows", overwrite = overwrt)
    
    # Paso 6: QR final de las R apiladas
    bdapply_Function_hdf5(filename = file,
                          group = "Step3/merged",
                          datasets = paste0(strdataset, "Rt"),
                          outgroup = "Step3/Final_QR",
                          func = "QR", overwrite = overwrt)
    
    # Paso 7: Dividir Q final en bloques
    bdSplit_matrix_hdf5(filename = file,
                        group = "Step3/Final_QR",
                        dataset = paste0(strdataset, "Rt.Q"),
                        outgroup = "Step4/splitted",
                        nblocks = m, bycols = bcols, overwrite = overwrt)
    
    # Paso 8: Multiplicar Q originales por piezas de Q final
    tmp <- bdgetDatasetsList_hdf5(file, "Step4/splitted")
    final_Q_pieces <- tmp[grep(paste0(strdataset, "Rt.Q"), tmp)]
    q_matrices <- blocks_qr[grep(".Q$", blocks_qr)]
    bdapply_Function_hdf5(filename = file,
                          group = paste0("Step2/", strdataset, "rows"),
                          datasets = q_matrices,
                          outgroup = "Step5", func = "blockmult",
                          b_group = "Step4/splitted", 
                          b_datasets = final_Q_pieces,
                          overwrite = TRUE)
    
    # Paso 9: Apilar resultados finales
    blocks_final <- bdgetDatasetsList_hdf5(file, "Step5")
    final_Q_blocks <- blocks_final[grep(paste0(strdataset, "."), blocks_final)]
    bdBind_hdf5_datasets(filename = file, group = "Step5",
                         datasets = final_Q_blocks,
                         outgroup = "Step6", 
                         outdataset = paste0(strdataset, "Q"),
                         func = "bindRows", overwrite = overwrt)
}

Explicación Detallada

Paso 1: Parsear el Path del Dataset

strgroup <- gsub("/.?$", "", strdataset)     # "data/X" → "data"
strdataset <- gsub("^.*/", "", strdataset)   # "data/X" → "X"

Qué hace: Separa un path como “data/X” en dos partes: - strgroup = "data" (el grupo, como una carpeta) - strdataset = "X" (el nombre del dataset, como un archivo)

Por qué es necesario: HDF5 organiza datos jerárquicamente (grupos dentro de grupos). Las funciones de BigDataStatMeth necesitan el grupo y el dataset por separado.

Ejemplo: Si pasas strdataset = "omics/genomics/SNPs": - strgroup = "omics/genomics" - strdataset = "SNPs"

Paso 2: Normalizar los Datos

bdNormalize_hdf5(filename = file, 
                 group = strgroup, 
                 dataset = strdataset,
                 bcenter = center,   # Centrar (restar media)
                 bscale = scale,     # Escalar (dividir por SD)
                 overwrite = overwrt)

Qué hace: Centra y/o escala la matriz: - Centrar (bcenter = TRUE): Resta la media de cada columna - Escalar (bscale = TRUE): Divide cada columna por su desviación estándar

Por qué es crítico para CCA: CCA asume datos centrados. Si no centras: - Las correlaciones canónicas están sesgadas - Los coeficientes canónicos son incorrectos - Los resultados no son interpretables

Dónde se guarda el resultado: Crea NORMALIZED/data/X en el archivo HDF5.

Nota: Para CCA, normalmente bcenter = TRUE y bscale = FALSE.

Paso 3: Dividir en Bloques

bdSplit_matrix_hdf5(filename = file,
                    group = paste0("NORMALIZED/", strgroup),
                    dataset = strdataset,
                    outgroup = paste0("Step1/", strdataset, "rows"),
                    nblocks = m,      # Número de bloques
                    bycols = bcols,   # FALSE = dividir por filas
                    overwrite = overwrt)

Qué hace: Divide la matriz normalizada en m bloques por filas.

Por qué dividir: Una matriz de 100,000 × 10,000 ocupa ~8GB en memoria. Si tienes 16GB de RAM, no puedes procesarla completa. Dividir en 10 bloques significa procesar 10,000 × 10,000 cada vez (~800MB).

Ejemplo visual:

Matriz X (100,000 filas)
    ↓ Dividir en 4 bloques
┌─────────────┐
│   X.1       │  25,000 filas
├─────────────┤
│   X.2       │  25,000 filas
├─────────────┤
│   X.3       │  25,000 filas
├─────────────┤
│   X.4       │  25,000 filas
└─────────────┘

Dónde se guardan: Step1/Xrows/X.1, Step1/Xrows/X.2, …, Step1/Xrows/X.m

Paso 4: Aplicar QR a Cada Bloque

blocks <- bdgetDatasetsList_hdf5(file, paste0("Step1/", strdataset, "rows"))

bdapply_Function_hdf5(filename = file,
                      group = paste0("Step1/", strdataset, "rows"),
                      datasets = blocks,
                      outgroup = paste0("Step2/", strdataset, "rows"),
                      func = "QR",    # Función a aplicar
                      overwrite = overwrt)

Qué hace: Para cada bloque X.i, calcula QR: X.i = Q.i × R.i

Por qué separar Q y R: - Q (ortogonal): Usaremos estas matrices más adelante - R (triangular superior): Apilaremos todas las R para un QR final

La magia de bdapply_Function_hdf5: Esta función aplica QR a todos los bloques automáticamente, sin escribir un bucle. Además, puede paralelizarse.

Resultado: Para cada bloque X.i, obtienes dos datasets: - Step2/Xrows/X.1.Q (parte ortogonal) - Step2/Xrows/X.1.R (parte triangular)

Paso 5: Apilar Matrices R

blocks_qr <- bdgetDatasetsList_hdf5(file, paste0("Step2/", strdataset, "rows"))
r_matrices <- blocks_qr[grep(".R$", blocks_qr)]  # Solo las R

bdBind_hdf5_datasets(filename = file,
                     group = paste0("Step2/", strdataset, "rows"),
                     datasets = r_matrices,
                     outgroup = "Step3/merged",
                     outdataset = paste0(strdataset, "Rt"),
                     func = "bindRows",  # Apilar verticalmente
                     overwrite = overwrt)

Qué hace: Apila todas las matrices R verticalmente: [R.1; R.2; R.3; R.4]

Por qué este paso es clave: Las matrices R son triangulares superiores y mucho más pequeñas que los bloques originales. Al apilarlas, obtenemos una matriz que SÍ cabe en memoria.

Ejemplo numérico:

Cada bloque: 25,000 × 10,000
Cada R:      10,000 × 10,000  (mucho más pequeña!)

R.1  (10,000 × 10,000)
R.2  (10,000 × 10,000)
R.3  (10,000 × 10,000)
R.4  (10,000 × 10,000)
        ↓ Apilar
R_stacked (40,000 × 10,000)

Ventaja: R_stacked es más pequeña que X original (100,000 × 10,000), así que podemos hacer QR de ella.

Resultado: Step3/merged/XRt contiene todas las R apiladas.

Paso 6: QR Final de las R Apiladas

bdapply_Function_hdf5(filename = file,
                      group = "Step3/merged",
                      datasets = paste0(strdataset, "Rt"),
                      outgroup = "Step3/Final_QR",
                      func = "QR",
                      overwrite = overwrt)

Qué hace: Calcula QR de R_stacked: R_stacked = Q_final × R_final

Insight matemático: Este Q_final y R_final representan la descomposición QR verdadera de la matriz X original. Pero todavía necesitamos combinarlos con las Q.i del Paso 4.

Por qué funciona esto: La propiedad de QR es que si X = Q × R, y R = Q’ × R’, entonces X = (Q × Q’) × R’. Estamos aplicando esta propiedad recursivamente.

Resultado: - Step3/Final_QR/XRt.Q (Q del QR final) - Step3/Final_QR/XRt.R (R del QR final - este es el R definitivo)

Paso 7: Dividir Q Final

bdSplit_matrix_hdf5(filename = file,
                    group = "Step3/Final_QR",
                    dataset = paste0(strdataset, "Rt.Q"),
                    outgroup = "Step4/splitted",
                    nblocks = m,
                    bycols = bcols, 
                    overwrite = overwrt)

Qué hace: Divide Q_final en m piezas (el mismo número que bloques originales).

Por qué: Vamos a multiplicar cada pieza de Q_final por su correspondiente Q.i del Paso 4.

Fórmula matemática: La Q verdadera es: Q = [Q.1 × Q_final_piece.1; Q.2 × Q_final_piece.2; …]

Resultado: Step4/splitted/XRt.Q.1, …, Step4/splitted/XRt.Q.m

Paso 8: Multiplicar Q Originales por Piezas de Q Final

tmp <- bdgetDatasetsList_hdf5(file, "Step4/splitted")
final_Q_pieces <- tmp[grep(paste0(strdataset, "Rt.Q"), tmp)]
q_matrices <- blocks_qr[grep(".Q$", blocks_qr)]  # Q del Paso 4

bdapply_Function_hdf5(filename = file,
                      group = paste0("Step2/", strdataset, "rows"),
                      datasets = q_matrices,
                      outgroup = "Step5", 
                      func = "blockmult",  # Multiplicación por bloques
                      b_group = "Step4/splitted", 
                      b_datasets = final_Q_pieces,
                      overwrite = TRUE)

Qué hace: Para cada bloque i, calcula: Q_verdadera.i = Q.i × Q_final_piece.i

Por qué este paso reconstruye la Q verdadera: - Las Q.i del Paso 4 son correctas para sus bloques locales - Q_final captura la estructura global - Multiplicarlas combina información local y global

Visualización:

Q.1 (25K × 25K) × Q_final_piece.1 (25K × 10K) = Q_true.1 (25K × 10K)
Q.2 (25K × 25K) × Q_final_piece.2 (25K × 10K) = Q_true.2 (25K × 10K)
...

Resultado: Step5/X.1, Step5/X.2, …, Step5/X.m (las Q verdaderas para cada bloque)

Paso 9: Apilar Q Finales

blocks_final <- bdgetDatasetsList_hdf5(file, "Step5")
final_Q_blocks <- blocks_final[grep(paste0(strdataset, "."), blocks_final)]

bdBind_hdf5_datasets(filename = file, 
                     group = "Step5",
                     datasets = final_Q_blocks,
                     outgroup = "Step6", 
                     outdataset = paste0(strdataset, "Q"),
                     func = "bindRows",  # Apilar verticalmente
                     overwrite = overwrt)
}

Qué hace: Apila todas las Q_true.i verticalmente para obtener la Q completa.

Resultado final: Step6/XQ contiene la matriz Q completa de la descomposición QR de X.

Lo que hemos logrado: Calcular QR de una matriz gigante (100,000 × 10,000) sin nunca cargarla completamente en memoria. Solo procesamos bloques de ~25,000 filas a la vez.

Part 2: Función Principal CCA

Código Completo

bdCCA_hdf5 <- function(filename, X, Y, m = 10,
                       bcenter = TRUE, bscale = FALSE, bycols = FALSE,
                       overwriteResults = FALSE, keepInteResults = FALSE) {
    
    # Paso 1-6: QR de X e Y usando helper function
    sapply(c(X, Y), getQRbyBlocks, 
           file = filename, m = m,
           center = bcenter, scale = bscale, 
           bcols = bycols, overwrt = overwriteResults)
    
    # Paso 7: Producto cruzado de matrices Q
    bdCrossprod_hdf5(filename = filename, 
                     group = "Step6", A = "XQ",
                     groupB = "Step6", B = "YQ",
                     outdataset = "CrossProd_XQ_YQ",
                     outgroup = "Step7", 
                     overwrite = TRUE)
    
    # Paso 8: SVD del producto cruzado
    bdSVD_hdf5(file = filename, 
               group = "Step7", 
               dataset = "CrossProd_XQ_YQ",
               bcenter = FALSE,  # Ya centrado en Paso 1
               bscale = FALSE, 
               k = 16,   # Primeras 16 correlaciones canónicas
               q = 2,    # Niveles jerárquicos para SVD
               threads = 3, 
               overwrite = TRUE)
    
    # Paso 9: Extraer componentes CCA
    res <- sapply(c(X, Y), bdgetDim_hdf5, filename = filename)
    writeCCAComponents_hdf5(filename, res[2, X], res[2, Y])
    
    # Paso 10: Limpiar resultados intermedios (opcional)
    if (!keepInteResults) {
        sapply(paste0("Step", 1:7), function(x) {
            invisible(bdRemove_hdf5_element(filename, element = x))
        })
    }
}

Explicación Detallada

Paso 1-6: QR para X e Y

sapply(c(X, Y), getQRbyBlocks, 
       file = filename, m = m,
       center = bcenter, scale = bscale, 
       bcols = bycols, overwrt = overwriteResults)

Qué hace: Llama a getQRbyBlocks() dos veces, una para X y otra para Y.

Por qué para ambas matrices: CCA necesita la descomposición QR de ambas matrices para computar las correlaciones canónicas.

Resultado después de este paso: - Step6/XQ - Matriz Q de X - Step6/YQ - Matriz Q de Y
- Step3/Final_QR/XRt.R - Matriz R de X - Step3/Final_QR/YRt.R - Matriz R de Y

Tiempo: Este es el paso más lento (puede tomar minutos para matrices grandes).

Paso 7: Producto Cruzado de Q Matrices

bdCrossprod_hdf5(filename = filename, 
                 group = "Step6", A = "XQ",
                 groupB = "Step6", B = "YQ",
                 outdataset = "CrossProd_XQ_YQ",
                 outgroup = "Step7", 
                 overwrite = TRUE)

Qué hace: Calcula C = t(Q_X) × Q_Y (transpuesta de Q_X multiplicada por Q_Y).

Por qué este producto específico: En CCA, las correlaciones canónicas provienen del SVD de esta matriz C. Este producto captura cómo se relacionan los espacios de columnas de X e Y.

Ventaja computacional: Q_X y Q_Y son mucho más pequeñas que X e Y originales, así que este producto es rápido incluso si las matrices originales eran enormes.

Tamaño de C: Si X tiene p columnas e Y tiene q columnas, C es p × q. Ejemplo: Si X es 100,000 × 50 e Y es 100,000 × 30, entonces C es solo 50 × 30.

Resultado: Step7/CrossProd_XQ_YQ - La matriz C que usaremos para SVD.

Paso 8: SVD del Producto Cruzado

bdSVD_hdf5(file = filename, 
           group = "Step7", 
           dataset = "CrossProd_XQ_YQ",
           bcenter = FALSE,  # Ya está centrado
           bscale = FALSE, 
           k = 16,   # Número de correlaciones a calcular
           q = 2,    # Niveles jerárquicos
           threads = 3, 
           overwrite = TRUE)

Qué hace: Calcula la Descomposición en Valores Singulares: C = U × D × V’

Qué significan U, D, V en el contexto de CCA: - D (valores singulares): Son las correlaciones canónicas - U (vectores singulares izquierdos): Usados para calcular coeficientes de X - V (vectores singulares derechos): Usados para calcular coeficientes de Y

Por qué k=16: Solo necesitamos las primeras correlaciones canónicas. Calcular todas sería innecesario (y más lento). Ajusta k según cuántas correlaciones necesites.

Qué es q=2: Niveles jerárquicos para SVD por bloques. Útil si C es muy grande. Para C pequeña (como 50 × 30), no importa mucho.

Resultado: - SVD/CrossProd_XQ_YQ/d - Vector con las correlaciones canónicas - SVD/CrossProd_XQ_YQ/u - Matriz U (p × k) - SVD/CrossProd_XQ_YQ/v - Matriz V (q × k)

Paso 9: Extraer Componentes CCA

res <- sapply(c(X, Y), bdgetDim_hdf5, filename = filename)
writeCCAComponents_hdf5(filename, res[2, X], res[2, Y])

Qué hace: 1. Obtiene las dimensiones de X e Y (necesitamos el número de columnas) 2. Llama a función helper que extrae coeficientes y scores canónicos

Por qué necesitamos las dimensiones: Las matrices Q tienen forma (n × n), pero solo necesitamos las primeras p o q columnas. Las dimensiones nos dicen cuántas columnas extraer.

Qué se calcula en writeCCAComponents_hdf5: (lo veremos en Part 3) - Coeficientes canónicos (cómo combinar variables originales) - Scores canónicos (valores de las variables canónicas) - Correlaciones canónicas

Paso 10: Limpiar Intermedios (Opcional)

if (!keepInteResults) {
    sapply(paste0("Step", 1:7), function(x) {
        invisible(bdRemove_hdf5_element(filename, element = x))
    })
}

Qué hace: Elimina los grupos Step1 a Step7 del archivo HDF5.

Por qué: Ahorra espacio en disco. Los resultados finales están en el grupo “Results”, así que los pasos intermedios ya no son necesarios.

Cuándo mantener intermedios (keepInteResults = TRUE): - Durante depuración (inspeccionar cada paso) - Si quieres verificar resultados intermedios - Si el análisis falló y quieres continuar desde un paso intermedio

Espacio ahorrado: Puede ser significativo (varios GB para matrices grandes).

Part 3: Extraer Resultados CCA

Código Completo

writeCCAComponents_hdf5 <- function(filename, ncolsX, ncolsY) {
    
    library(rhdf5)
    h5f <- H5Fopen(filename)
    
    # Leer Q, R y resultados de SVD
    XQ <- h5f$Step6$XQ[1:ncolsX, 1:ncolsX]
    YQ <- h5f$Step6$YQ[1:ncolsY, 1:ncolsY]
    XR <- h5f$Step3$Final_QR$XRt.R
    YR <- h5f$Step3$Final_QR$YRt.R
    d <- h5f$SVD$CrossProd_XQ_YQ$d
    u <- h5f$SVD$CrossProd_XQ_YQ$u
    v <- h5f$SVD$CrossProd_XQ_YQ$v
    xcenter <- h5f$NORMALIZED$data$mean.X
    ycenter <- h5f$NORMALIZED$data$mean.Y
    x.names <- h5f$data$.X_dimnames$`2`
    y.names <- h5f$data$.Y_dimnames$`2`
    
    H5Fclose(h5f)
    
    # Reconstruir QR compacto
    XR[lower.tri(XR, diag = FALSE)] <- 0
    XQ[upper.tri(XQ, diag = TRUE)] <- 0
    XQR <- XR + XQ
    
    YR[lower.tri(YR, diag = FALSE)] <- 0
    YQ[upper.tri(YQ, diag = TRUE)] <- 0
    YQR <- YR + YQ
    
    # Calcular coeficientes canónicos
    xcoef <- bdSolve(XQR, u)
    ycoef <- bdSolve(YQR, v)
    rownames(xcoef) <- as.matrix(x.names)
    rownames(ycoef) <- as.matrix(y.names)
    
    # Guardar resultados
    bdCreate_hdf5_matrix(filename, xcoef, "Results", "xcoef", overwriteDataset = TRUE)
    bdCreate_hdf5_matrix(filename, ycoef, "Results", "ycoef", overwriteDataset = TRUE)
    bdCreate_hdf5_matrix(filename, as.matrix(diag(d)), "Results", "cor", overwriteDataset = TRUE)
    bdCreate_hdf5_matrix(filename, xcenter, "Results", "xcenter", overwriteDataset = TRUE)
    bdCreate_hdf5_matrix(filename, ycenter, "Results", "ycenter", overwriteDataset = TRUE)
    
    # Calcular scores canónicos
    bdblockmult_hdf5(filename, "data", "X", "xcoef", 
                     groupB = "Results", outgroup = "Results",
                     outdataset = "xscores", overwrite = TRUE)
    bdblockmult_hdf5(filename, "data", "Y", "ycoef", 
                     groupB = "Results", outgroup = "Results",
                     outdataset = "yscores", overwrite = TRUE)
}

Explicación Detallada

Leer Datos del HDF5

library(rhdf5)
h5f <- H5Fopen(filename)

XQ <- h5f$Step6$XQ[1:ncolsX, 1:ncolsX]
YQ <- h5f$Step6$YQ[1:ncolsY, 1:ncolsY]
XR <- h5f$Step3$Final_QR$XRt.R
YR <- h5f$Step3$Final_QR$YRt.R
d <- h5f$SVD$CrossProd_XQ_YQ$d
u <- h5f$SVD$CrossProd_XQ_YQ$u
v <- h5f$SVD$CrossProd_XQ_YQ$v

Qué hace: Abre el archivo HDF5 y lee las matrices que necesitamos.

Por qué [1:ncolsX, 1:ncolsX] para Q: Las matrices Q tienen tamaño (n × n) donde n es el número de filas. Pero solo necesitamos las primeras p columnas (número de variables en X).

Qué datos leemos: - Q matrices: Del Paso 6 (resultado del QR por bloques) - R matrices: Del Paso 6 (parte triangular del QR) - d, u, v: Del SVD (Paso 8) - Medias: Para poder “descentrar” si es necesario

Reconstruir QR Compacto

XR[lower.tri(XR, diag = FALSE)] <- 0  # Poner ceros debajo diagonal
XQ[upper.tri(XQ, diag = TRUE)] <- 0   # Poner ceros arriba diagonal
XQR <- XR + XQ

Qué hace: Combina Q y R en una sola matriz (formato compacto).

Por qué este formato: La descomposición QR se puede almacenar de forma compacta: - Parte inferior (incluyendo diagonal): Q - Parte superior (excluyendo diagonal): R

Visual:

XQR compacto:
┌─────────────┐
│ R  R  R  R  │  ← Parte superior: R
│ Q  R  R  R  │
│ Q  Q  R  R  │
│ Q  Q  Q  R  │
│ Q  Q  Q  Q  │  ← Parte inferior: Q
└─────────────┘

Para qué lo necesitamos: Resolver el sistema R × coef = u

Calcular Coeficientes Canónicos

xcoef <- bdSolve(XQR, u)
ycoef <- bdSolve(YQR, v)
rownames(xcoef) <- as.matrix(x.names)
rownames(ycoef) <- as.matrix(y.names)

Qué hace: Resuelve los sistemas lineales: - R_X × xcoef = u - R_Y × ycoef = v

Qué son los coeficientes canónicos: Los pesos para combinar las variables originales. Si X tiene 50 variables, xcoef te dice cuánto peso darle a cada una para formar las variables canónicas.

Interpretación: Si xcoef[5, 1] = 0.8, significa que la variable 5 de X contribuye fuertemente (con peso 0.8) a la primera variable canónica.

Por qué resolver este sistema: Matemáticamente, las variables canónicas son X × xcoef. Como X = Q_X × R_X, y ya tenemos Q_X × u del SVD, necesitamos xcoef = solve(R_X, u).

Por qué añadir nombres: Hace que los resultados sean interpretables. En lugar de “variable 5”, verás “SNP_rs12345” o el nombre real de tu variable.

Guardar Resultados en HDF5

bdCreate_hdf5_matrix(filename, ycoef, "Results", "ycoef", overwriteDataset = TRUE)
bdCreate_hdf5_matrix(filename, as.matrix(diag(d)), "Results", "cor", overwriteDataset = TRUE)
bdCreate_hdf5_matrix(filename, xcenter, "Results", "xcenter", overwriteDataset = TRUE)
bdCreate_hdf5_matrix(filename, ycenter, "Results", "ycenter", overwriteDataset = TRUE)

Qué guarda en el grupo “Results”:

Dataset Contenido Dimensiones
xcoef Coeficientes canónicos de X p × k
ycoef Coeficientes canónicos de Y q × k
cor Correlaciones canónicas k × 1
xcenter Medias de columnas de X p × 1
ycenter Medias de columnas de Y q × 1

Donde: - p = número de variables en X - q = número de variables en Y - k = número de correlaciones canónicas calculadas (16 en nuestro caso)

Por qué guardar las medias: Por si quieres reconstruir las variables originales a partir de las variables canónicas.

Calcular Scores Canónicos

bdblockmult_hdf5(filename, "data", "X", "xcoef", 
                 groupB = "Results", outgroup = "Results",
                 outdataset = "xscores", overwrite = TRUE)

bdblockmult_hdf5(filename, "data", "Y", "ycoef", 
                 groupB = "Results", outgroup = "Results",
                 outdataset = "yscores", overwrite = TRUE)

Qué hace: Calcula las variables canónicas para cada muestra: - xscores = X × xcoef - yscores = Y × ycoef

Qué son los scores canónicos: Los valores de las nuevas variables canónicas para cada observación. Si tienes 1000 muestras y k=16 correlaciones, xscores será 1000 × 16.

Para qué se usan: - Visualización: Plotear xscores[,1] vs yscores[,1] para ver la primera correlación canónica - Análisis: Los scores son las variables transformadas que puedes usar en análisis posteriores - Interpretación: Ver qué muestras tienen valores altos/bajos en cada variable canónica

Por qué usar bdblockmult_hdf5: Si X es muy grande (100,000 filas), esta función multiplica por bloques sin cargar todo X en memoria.

Ejemplo de Uso Completo

library(BigDataStatMeth)

# Crear datos sintéticos con estructura compartida
set.seed(123)
n <- 1000  # Muestras
latent <- matrix(rnorm(n * 5), n, 5)  # 5 factores latentes compartidos

# X e Y comparten estos factores latentes
X <- latent %*% matrix(rnorm(5 * 50), 5, 50) + matrix(rnorm(n * 50, sd = 0.5), n, 50)
Y <- latent %*% matrix(rnorm(5 * 30), 5, 30) + matrix(rnorm(n * 30, sd = 0.5), n, 30)

Por qué estos datos: Creamos X e Y con 5 factores latentes compartidos. CCA debería detectar estos 5 factores, resultando en 5 correlaciones canónicas altas.

# Guardar en HDF5
filename <- "cca_example.h5"
bdCreate_hdf5_matrix(filename, X, "data", "X", overwriteFile = TRUE)
bdCreate_hdf5_matrix(filename, Y, "data", "Y")

Nota: overwriteFile = TRUE en la primera creación para borrar cualquier archivo previo. Luego FALSE (por defecto) para añadir Y al mismo archivo.

# Ejecutar CCA
bdCCA_hdf5(filename, "data/X", "data/Y", m = 4, bcenter = TRUE, bscale = FALSE)

Parámetros usados: - m = 4: Dividir matrices en 4 bloques - bcenter = TRUE: Centrar datos (necesario para CCA) - bscale = FALSE: No escalar (típico en CCA)

Tiempo de ejecución: Para este ejemplo (~30 segundos). Para matrices reales grandes puede ser minutos.

Patrón de Composición

CCA se construyó componiendo funciones simples:

bdNormalize_hdf5()      → Centrar datos
    ↓
bdSplit_matrix_hdf5()   → Dividir en bloques
    ↓
bdapply_Function_hdf5() → Aplicar QR a bloques
    ↓
bdBind_hdf5_datasets()  → Combinar resultados
    ↓
bdCrossprod_hdf5()      → Producto cruzado
    ↓
bdSVD_hdf5()            → Descomposición
    ↓
Resultados

Este mismo patrón funciona para cualquier método complejo.

Estrategia Block-wise

Problema: Matriz demasiado grande
    ↓
Solución: Dividir → Procesar → Combinar → Refinar
    ↓
Resultado: QR de matriz gigante sin cargarla en memoria

Almacenamiento Intermedio

Guardar resultados de cada paso en HDF5: - ✅ Debugging fácil (inspeccionar cualquier paso) - ✅ Recuperación ante fallos (continuar desde paso intermedio) - ✅ Entendimiento (ver qué produce cada paso)

Aplicabilidad

Estos patrones se aplican a: - Partial Least Squares (PLS): Similar a CCA - Principal Component Regression (PCR): PCA + regresión - Independent Component Analysis (ICA): Descomposición iterativa - Tu método personalizado: Cualquier análisis multi-paso
