Installation

System Requirements

Before installing BigDataStatMeth, ensure your system meets these requirements:

R Requirements

  • R version ≥ 4.0.0
  • Operating System: Linux, macOS, or Windows

Compiler Tools

BigDataStatMeth includes C++ code that needs compilation:

  • Windows
  • macOS
  • Linux (Ubuntu/Debian)

Install Rtools matching your R version:

# Check your R version
R.version.string

# Download and install appropriate Rtools from:
# https://cran.r-project.org/bin/windows/Rtools/

Install Xcode Command Line Tools:

xcode-select --install
sudo apt-get update
sudo apt-get install build-essential

HDF5 Libraries

For working with HDF5 files (required for large datasets):

  • Windows
  • macOS
  • Linux (Ubuntu/Debian)

No additional installation needed. The rhdf5 Bioconductor package includes HDF5 libraries.

# Using Homebrew
brew install hdf5

# If compilation fails later, set HDF5 path:
export HDF5_DIR=/opt/homebrew/opt/hdf5
sudo apt-get install libhdf5-dev

Installing BigDataStatMeth

From GitHub (Recommended)

# Install devtools if not already available
if (!requireNamespace("devtools", quietly = TRUE)) {
  install.packages("devtools")
}

# Install BigDataStatMeth from GitHub
devtools::install_github("isglobal-brge/BigDataStatMeth@HEAD")

Dependencies

The following packages will be installed automatically:

CRAN packages: - Rcpp, RcppEigen - C++ integration - Matrix - Sparse matrix support

Bioconductor packages: - rhdf5 - HDF5 file format support

Verify Installation

After installation, run these checks to ensure everything works:

Test 1: Load Package

library(BigDataStatMeth)

# Check version
packageVersion("BigDataStatMeth")
# Should display version number (e.g., 1.0.0)

Test 2: Basic Matrix Operations

# Create test matrices
set.seed(123)
X <- matrix(rnorm(100), nrow = 10, ncol = 10)
Y <- matrix(rnorm(100), nrow = 10, ncol = 10)

# Test block multiplication
result <- bdblockmult(X, Y, blocksize = 5)

# Verify result
expected <- X %*% Y
all.equal(result, expected, tolerance = 1e-10)
# Should return TRUE

✅ If TRUE: Basic operations working correctly.

Test 3: HDF5 Support

library(rhdf5)

# Create temporary HDF5 file
test_file <- tempfile(fileext = ".h5")

# Test HDF5 write
bdCreate_hdf5_matrix(
  filename = test_file,
  object = X,
  group = "test",
  dataset = "matrix_X",
  overwriteFile = TRUE
)

# Test HDF5 read
h5ls(test_file)
# Should show: group 'test', dataset 'matrix_X'

# Cleanup
file.remove(test_file)

✅ If no errors: HDF5 functionality working.

Test 4: Parallel Computing

# Test parallel block multiplication
result_parallel <- bdblockmult(
  X, Y,
  blocksize = 5,
  paral = TRUE
)

all.equal(result, result_parallel)
# Should return TRUE

✅ If TRUE: Parallel computing enabled.

Troubleshooting

Issue 1: Compilation Errors

Symptoms: Installation fails with compilation errors.

Solutions:

  1. Update R and compiler tools:

    # Check R version
R.version.string
# Update R if < 4.0.0

# Windows: Update Rtools
# macOS: xcode-select --install
# Linux: sudo apt-get install build-essential

  2. Install dependencies manually:

    install.packages(c("Rcpp", "RcppEigen", "Matrix"))
BiocManager::install("rhdf5")

  3. Clean and retry:

    remove.packages("BigDataStatMeth")
devtools::install_github("isglobal-brge/BigDataStatMeth@HEAD")

Issue 2: HDF5 Library Not Found

Symptoms: Error messages like:

cannot find -lhdf5
/usr/bin/ld: cannot find -lhdf5_hl

Solutions:

  • macOS
  • Linux
# Install HDF5 via Homebrew
brew install hdf5

# Set HDF5 path before installing package
export HDF5_DIR=/opt/homebrew/opt/hdf5

# Reinstall rhdf5
R -e 'BiocManager::install("rhdf5", force = TRUE)'
# Install development libraries
sudo apt-get install libhdf5-dev

# Verify installation
dpkg -l | grep hdf5

# Reinstall rhdf5
R -e 'BiocManager::install("rhdf5", force = TRUE)'

Issue 3: Rcpp Compilation Warnings

Symptoms: Many compiler warnings during installation.

Solution: These are usually harmless. Verify the package loads:

library(BigDataStatMeth)
# If no errors, installation successful despite warnings

Issue 4: Parallel Functions Not Working

Symptoms: paral = TRUE has no effect or causes errors.

Solutions:

  1. Check parallel package:

    library(parallel)
detectCores()  # Should show > 1

  2. Test with explicit thread count:

    bdblockmult(X, Y, blocksize = 5, paral = TRUE, threads = 2)

  3. Update parallel package:

    install.packages("parallel")

Complete Installation Check

Run this comprehensive test:

# Installation verification script
test_installation <- function() {
  cat("=== BigDataStatMeth Installation Check ===\n\n")
  
  # Test 1: Package loads
  cat("[1/4] Loading package... ")
  tryCatch({
    library(BigDataStatMeth, quietly = TRUE)
    cat("✓\n")
  }, error = function(e) {
    cat("✗\n")
    stop("Package failed to load: ", e$message)
  })
  
  # Test 2: Matrix operations
  cat("[2/4] Testing matrix operations... ")
  X <- matrix(rnorm(100), 10, 10)
  Y <- matrix(rnorm(100), 10, 10)
  result <- bdblockmult(X, Y, blocksize = 5)
  if (isTRUE(all.equal(result, X %*% Y))) {
    cat("✓\n")
  } else {
    cat("✗\n")
    warning("Matrix operations test failed")
  }
  
  # Test 3: HDF5 support
  cat("[3/4] Testing HDF5 support... ")
  tryCatch({
    test_file <- tempfile(fileext = ".h5")
    bdCreate_hdf5_matrix(test_file, X, "test", "X", overwriteFile = TRUE)
    if (file.exists(test_file)) {
      file.remove(test_file)
      cat("✓\n")
    }
  }, error = function(e) {
    cat("✗\n")
    warning("HDF5 test failed: ", e$message)
  })
  
  # Test 4: Parallel computing
  cat("[4/4] Testing parallel computing... ")
  tryCatch({
    result_par <- bdblockmult(X, Y, blocksize = 5, paral = TRUE)
    if (isTRUE(all.equal(result, result_par))) {
      cat("✓\n")
    }
  }, error = function(e) {
    cat("✗\n")
    warning("Parallel computing test failed: ", e$message)
  })
  
  cat("\n=== Installation Check Complete ===\n")
  cat("Version: ", as.character(packageVersion("BigDataStatMeth")), "\n")
}

# Run the test
test_installation()

Expected output:

=== BigDataStatMeth Installation Check ===

[1/4] Loading package... ✓
[2/4] Testing matrix operations... ✓
[3/4] Testing HDF5 support... ✓
[4/4] Testing parallel computing... ✓

=== Installation Check Complete ===
Version:  1.0.0

Getting Help

If problems persist after trying these solutions:

  1. Check existing issues: GitHub Issues

  2. Create a new issue with:

    • Your R version: sessionInfo()
    • Operating system and version
    • Complete error message
    • Minimal reproducible example

  3. Session information:

    sessionInfo()
