Example: End-to-End Analysis

Hereâ€™s a complete workflow from data to visualization:

library(BigDataStatMeth)

# 1. Create test data
set.seed(123)
data_matrix <- matrix(rnorm(5000), nrow = 1000, ncol = 5)

# 2. Save to HDF5
filename <- "example.h5"
bdCreate_hdf5_matrix(
  filename = filename,
  object = data_matrix,
  group = "data",
  dataset = "X",
  overwriteFile = TRUE
)

# 3. Normalize data
bdNormalize_hdf5(
  filename = filename,
  group = "data",
  dataset = "X",
  bcenter = TRUE,
  bscale = TRUE
)

# 4. Perform PCA
bdPCA_hdf5(
  filename = filename,
  group = "NORMALIZED/data",
  dataset = "X",
  k = 3,
  bcenter = FALSE,  # Already centered
  bscale = FALSE
)

# 5. Visualize results
library(rhdf5)
scores <- h5read(filename, "/PCA/X/u")

plot(scores[, 1], scores[, 2],
     xlab = "PC1", ylab = "PC2",
     main = "PCA Results",
     pch = 19, col = "steelblue")

# Cleanup
h5closeAll()
file.remove(filename)

Output: A PCA plot showing the structure in your data.

Package Philosophy

Design Principles

Composability: Small, focused functions that work together
Consistency: Similar APIs across all operations
Performance: Optimized C++ core with clean R interface
Flexibility: Work in memory or on disk
Extensibility: Easy to build new methods

Why This Matters

Traditional packages offer fixed workflows. BigDataStatMeth gives you building blocks:

# Build your own method by composing operations
custom_method <- function(file, data) {
    bdNormalize_hdf5(file, "data", data)
    bdSplit_matrix_hdf5(file, "NORMALIZED/data", data, nblocks = 10)
    blocks <- bdgetDatasetsList_hdf5(file, "NORMALIZED/data")
    bdapply_Function_hdf5(file, "NORMALIZED/data", blocks, func = "QR")
    bdBind_hdf5_datasets(file, "results", blocks, func = "bindRows")
}

This enables: - Custom preprocessing pipelines - Novel statistical methods - Domain-specific adaptations - Rapid prototyping
