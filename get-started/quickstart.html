Three Ways to Use BigDataStatMeth

BigDataStatMeth offers three approaches depending on your needs:

ðŸŽ¯ Way 1: Use Existing Methods

Who: Researchers, data analysts
What: Call ready-made statistical functions

library(BigDataStatMeth)

# Perform PCA on large dataset
bdPCA_hdf5(
  filename = "data.h5",
  group = "matrices",
  dataset = "X",
  k = 10,
  bcenter = TRUE
)

Best for: Standard analyses, getting results quickly.

ðŸ”§ Way 2: Develop Methods in R

Who: Statisticians, method developers
What: Combine BigDataStatMeth functions to create new methods

# Example: Custom preprocessing workflow
my_analysis <- function(filename, dataset) {
    # Step 1: Normalize
    bdNormalize_hdf5(filename, "data", dataset, bcenter = TRUE)
    
    # Step 2: Split into blocks
    bdSplit_matrix_hdf5(filename, "NORMALIZED/data", dataset, nblocks = 10)
    
    # Step 3: Apply QR to each block
    blocks <- bdgetDatasetsList_hdf5(filename, "NORMALIZED/data")
    bdapply_Function_hdf5(filename, "NORMALIZED/data", blocks, func = "QR")
    
    # Step 4: Combine results
    bdBind_hdf5_datasets(filename, "results", blocks, func = "bindRows")
}

Best for: Custom algorithms, domain-specific methods, prototyping.

âš¡ Way 3: Develop Methods in C++

Who: Developers needing maximum performance
What: Use BigDataStatMethâ€™s header-only C++ API

// [[Rcpp::depends(BigDataStatMeth)]]
#include "BigDataStatMeth.hpp"

// [[Rcpp::export]]
Rcpp::NumericVector my_fast_method(std::string filename) {
    // Use C++ classes instead of raw HDF5 C API
    hdf5Dataset ds(filename, "/data/matrix");
    
    // Easy block reading
    Eigen::MatrixXd block = ds.readBlock(0, 0, 1000, 100);
    
    // Your algorithm here...
    Eigen::VectorXd result = block.colwise().sum();
    
    return Rcpp::wrap(result);
}

Why this matters: You avoid writing 50+ lines of verbose HDF5 C API code. BigDataStatMeth provides clean, high-level classes.

Best for: Performance-critical code, iterative algorithms, production systems.

Your First Example

Letâ€™s walk through a simple analysis using both in-memory and HDF5 modes.

Setup

library(BigDataStatMeth)

# Create test data
set.seed(123)
X <- matrix(rnorm(1000 * 500), nrow = 1000, ncol = 500)
Y <- matrix(rnorm(1000 * 500), nrow = 1000, ncol = 500)

In-Memory Mode

For data that fits in RAM:

# Matrix multiplication
result <- bdblockmult(X, Y, blocksize = 128, paral = TRUE)

# Cross product: t(X) %*% X
XtX <- bdCrossprod(X, blocksize = 128)

# Transpose product: X %*% t(X)
XXt <- bdtCrossprod(X, blocksize = 128)

HDF5 Mode

For large datasets on disk:

# 1. Create HDF5 file
filename <- "mydata.h5"

bdCreate_hdf5_matrix(
  filename = filename,
  object = X,
  group = "matrices",
  dataset = "X",
  overwriteFile = TRUE
)

bdCreate_hdf5_matrix(
  filename = filename,
  object = Y,
  group = "matrices",
  dataset = "Y"
)

# 2. Operations on disk (same syntax!)
result_hdf5 <- bdblockmult_hdf5(
  filename = filename,
  group = "matrices",
  A = "X",
  B = "Y"
)

# 3. Cross products
bdCrossprod_hdf5(
  filename = filename,
  group = "matrices",
  A = "X"
)

Key insight: Almost identical syntax. Just add _hdf5 and pass file paths instead of matrices.

When to Use Each Mode

Criterion In-Memory HDF5
Data size < 5 GB > 5 GB
Available RAM > 2Ã— data size Any
Speed Fastest Very fast
Workflow Simple Multi-step pipelines

Rule of thumb: If your matrix is larger than half your available RAM, use HDF5 mode.

Common Operations

Matrix Multiplication

# In-memory
C <- bdblockmult(A, B, blocksize = 128, paral = TRUE)

# HDF5
bdblockmult_hdf5(filename, "data", "A", "B")

Cross Products

Cross products are fundamental for many statistical methods:

# t(X) %*% X (covariance-like structure)
XtX <- bdCrossprod_hdf5(filename, "data", "X")

# X %*% t(X) (Gram matrix)
XXt <- bdtCrossprod_hdf5(filename, "data", "X")

Used in: Covariance matrices, correlation, linear regression, PCA.

Normalization

# Center and scale data
bdNormalize_hdf5(
  filename = filename,
  group = "data",
  dataset = "X",
  bcenter = TRUE,   # Subtract column means
  bscale = TRUE     # Divide by standard deviation
)

Principal Component Analysis

# Perform PCA
pca <- bdPCA_hdf5(
  filename = filename,
  group = "data",
  dataset = "X",
  k = 10,           # Number of components
  bcenter = TRUE,
  bscale = FALSE,
  threads = 4       # Parallel threads
)

# Results stored in HDF5 file
library(rhdf5)
h5ls(filename)  # See what was created

# Access results
loadings <- h5read(filename, "/PCA/X/v")  # Eigenvectors
scores <- h5read(filename, "/PCA/X/u")    # Principal components
variance <- h5read(filename, "/PCA/X/d")  # Singular values

Parallel Computing

Most functions support parallelization:

# Enable parallel processing
result <- bdblockmult(
  X, Y,
  blocksize = 128,
  paral = TRUE,
  threads = 4      # Or NULL to auto-detect
)

# Check available cores
parallel::detectCores()

Performance tip: Parallelization helps most with large matrices (> 5000 Ã— 5000) and computationally intensive operations (SVD, PCA).

Choosing Block Size

Block size affects performance and memory usage:

# Small blocks: less memory, more overhead
bdblockmult(X, Y, blocksize = 64)

# Large blocks: more memory, less overhead
bdblockmult(X, Y, blocksize = 512)

# Default (recommended)
bdblockmult(X, Y, blocksize = 128)

Guidelines: - RAM abundant: Use larger blocks (512, 1024) - RAM limited: Use smaller blocks (64, 128) - Unsure: Start with 128

Complete Example

From start to finish:

library(BigDataStatMeth)

# 1. Prepare data
set.seed(42)
data_matrix <- matrix(rnorm(10000), nrow = 1000, ncol = 10)

# 2. Create HDF5 file
filename <- "analysis.h5"
bdCreate_hdf5_matrix(
  filename, data_matrix, 
  "data", "X", 
  overwriteFile = TRUE
)

# 3. Run PCA
bdPCA_hdf5(filename, "data", "X", k = 5, bcenter = TRUE)

# 4. Visualize results
library(rhdf5)
scores <- h5read(filename, "/PCA/X/u")

plot(scores[, 1], scores[, 2],
     xlab = "PC1", ylab = "PC2",
     main = "PCA Results",
     pch = 19, col = "steelblue")

# 5. Cleanup
h5closeAll()
